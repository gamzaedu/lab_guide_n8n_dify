### 시나리오

> 당신은 부서 업무추진비·출장비 영수증을 정리하는 담당자입니다.
> 매달 수십 장의 영수증 이미지를 한 장씩 엑셀에 옮겨 적는 데 반나절이 걸립니다.
> 영수증 이미지를 Webhook(또는 Google Form)으로 올리면, n8n이 GPT Vision으로 날짜·금액·상호·항목을 추출해 Google Sheets의 지출 대장에 자동 기록하도록 만드세요.

### 최종 산출물

- 작동하는 n8n 워크플로우 (Webhook 업로드 → 이미지 처리 → GPT Vision → Sheets 기록)
- 지출 대장 샘플 Google Sheet (최소 3건 자동 기록 완료)
- 추출 프롬프트와 필드 스키마 정의 메모

## Phase 1. 정의하기 (Define) — 약 20분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 Webhook 바이너리 수신과 Sheets 쓰기를 바탕으로, 지금부터는 스스로 "어떤 정보를 어디에 저장할지"를 정의합니다.

!!! abstract "과제"
    지출 대장에 기록해야 할 필수 컬럼 5~7개를 정의하고, 각 컬럼을 영수증의 어느 부분에서 어떻게 추출할지 규칙을 적으세요.

**요구사항 정리**

- [ ] 대장 컬럼 5~7개가 정의되었다 (예: 일자, 상호, 금액, 항목, 결제수단, 부가세, 비고)
- [ ] 각 컬럼의 형식이 정해졌다 (날짜 = `YYYY-MM-DD`, 금액 = 숫자만)
- [ ] 추출이 불가능하거나 불확실한 경우의 처리 규칙이 정해졌다 ("빈 칸" / "검토 필요")
- [ ] 업로드 방식(Webhook 파일 첨부 / Google Form / 이메일)이 결정되었다

**활용 도구 & 데이터**

- 샘플 영수증 이미지 3~5장 (각자 본인 영수증이나 검색으로 확보)
- Google Sheets: 지출 대장 템플릿 미리 생성 (헤더 행만)
- ChatGPT: 필드 스키마 점검, 프롬프트 초안 작성

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    실제 월말 지출 정산 엑셀을 한 장 열어보세요. "이 엑셀을 그대로 자동으로 채우려면 영수증에서 뭘 읽어야 하나?"가 바로 요구사항의 시작점입니다.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에게 다음과 같이 물어보세요:
    "공공기관 업무추진비 지출 대장에 기록하는 필수 컬럼과 각 컬럼의 형식(날짜/통화/문자열)을 JSON 스키마로 작성해줘. 영수증 이미지에서 못 읽을 수 있는 항목은 표시도 해줘."

    나온 스키마를 실제 영수증 사진에 대조해 현실성을 확인하세요.

## Phase 2. 구현하기 (Build) — 약 55분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    Webhook으로 영수증 이미지를 받아 GPT Vision으로 필드를 추출하고, Google Sheets에 한 행으로 기록하는 워크플로우를 만드세요. 최소 3장의 실제 영수증으로 end-to-end 테스트를 성공시킵니다.

**마일스톤**

☐ Webhook 노드가 이미지 바이너리 파일을 정상 수신함을 확인했다
☐ Edit Fields / Code로 OpenAI Vision에 전달할 포맷(Base64 또는 URL)으로 변환했다
☐ OpenAI Vision 노드가 JSON 형식으로 필드를 반환했다 (일자/금액/상호 포함)
☐ 반환된 JSON을 Sheets 컬럼 순서에 맞게 매핑했다
☐ Google Sheets에 실제로 한 행이 추가됐다
☐ 서로 다른 영수증 3장으로 반복 테스트해 모두 기록이 성공했다

**활용 도구 & 데이터**

- n8n: Webhook / Edit Fields / OpenAI (Vision 모델) / Google Sheets 노드
- Google Sheets: 대장 템플릿 (헤더 행 작성 후 공유)
- ChatGPT: Vision 프롬프트 반복 개선

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    먼저 Webhook URL을 받은 뒤, 브라우저나 Postman으로 이미지 한 장만 보내 수신 자체를 검증하세요.
    다음에 OpenAI 노드로 "이 이미지에 무엇이 보이는가?"만 물어봐 연결을 확인합니다.
    마지막에 프롬프트를 구조화해 JSON으로 받고, Sheets로 보냅니다.

??? tip "힌트 2: 도구 활용이 막힌다면"
    GPT가 JSON이 아닌 일반 문장으로 답하면 Sheets 매핑이 깨집니다. 프롬프트 끝에 반드시 명시하세요.
    ```
    반드시 아래 JSON 스키마로만 답하라. 설명 문장을 덧붙이지 마라.
    {
      "date": "YYYY-MM-DD",
      "vendor": "string",
      "amount": number,
      "category": "string",
      "vat": number,
      "payment_method": "string",
      "confidence": "high|medium|low"
    }
    ```

    그리고 OpenAI 노드의 `Response Format`을 `JSON`으로 설정하면 더 안정적입니다.

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    Sheets 매핑 예:
    - `A열 일자` ← `{{ $json.date }}`
    - `B열 상호` ← `{{ $json.vendor }}`
    - `C열 금액` ← `{{ $json.amount }}`
    - `D열 항목` ← `{{ $json.category }}`
    - `E열 신뢰도` ← `{{ $json.confidence }}`

    신뢰도가 `low`인 행은 색상(조건부 서식)으로 강조해두면 사람이 검수할 때 편합니다.

## Phase 3. 확장하기 (Extend) — 약 25분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    다양한 조건의 영수증 5장(흐릿한 것, 손글씨, 외국어 포함 등)을 돌려보고 실패 사례 2건을 찾아 원인을 분석·개선하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    실패 원인은 보통 네 가지 중 하나입니다.

    - 이미지 품질(해상도·기울기) → 전처리 안내 추가 또는 재촬영 요청
    - 프롬프트 모호 → "금액은 부가세 포함 총액" 같은 규칙 명시
    - 모델 선택 → 더 강력한 Vision 모델로 교체
    - 스키마 누락 → "읽을 수 없으면 null"처럼 예외 규칙 추가

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    다음 중 하나를 추가하세요. (1) 월별 합계를 자동 집계해 팀장에게 월말 메일 발송, (2) 금액이 특정 기준을 넘으면 Slack 알림, (3) 같은 구조로 "명함 → 연락처 대장" 버전 복제.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    이 워크플로우의 본질은 "이미지 → 구조화 데이터 → 표"입니다. 같은 뼈대에서 이미지 종류와 스키마만 바뀌면 됩니다.
    명함·출입증·민원 접수 양식 등 부서 내 반복 업로드 서식을 떠올려보세요.
