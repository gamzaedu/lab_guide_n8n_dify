# Project 04. 법령·조례 해석 도우미

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 법령·조례 해석 도우미 |
| 소요 시간 | 2교시 (100분) |
| 선행 Lab | **Lab 04 · 지식베이스 만들기**, **Lab 05 · 내부 규정 QA 봇**, **Lab 06 · 검색 설정 최적화** |
| 사용 도구 | Dify Cloud (Chatbot + Knowledge), ChatGPT, 법령 원문 |

📌 **진행 흐름**

`Phase 1 정의하기 (약 20분)` → `Phase 2 구현하기 (약 55분)` → `Phase 3 확장하기 (약 25분)`

> 이 프로젝트는 **Lab 04 · 지식베이스 만들기**, **Lab 05 · 내부 규정 QA 봇**, **Lab 06 · 검색 설정 최적화**에서 익힌 지식베이스 구축과 검색 튜닝을 법령 원문이라는 **전문 용어 중심 문서**에 적용합니다.

### 시나리오

> 당신은 법무·기획 담당자입니다.
> 특정 업무 상황에서 적용되는 법령·조례를 빠르게 해석해야 하는데, 조문 용어가 어려워 일반 공무원에게 설명하기 쉽지 않습니다. 당신은 대상 법령 원문을 기반으로 질문에 **원문 인용 + 평이한 해석 + 주의사항**을 함께 답해주는 도우미를 만들어, 부서 공용 또는 기관 내부 1차 자료로 활용하려 합니다. 단, 최종 법적 판단은 전문가가 해야 하므로 "법률 자문이 아님" 고지가 필수입니다.

### 최종 산출물

- 법령·조례 원문이 조문 단위로 청킹된 지식베이스
- 지식베이스와 연결된 Chatbot 앱 (법률 자문 아님 고지 포함)
- 질문 유형 3종(용어 해석 · 조문 적용 · 사례 질의)에 각 3건씩 9건 테스트 결과

## Phase 1. 정의하기 (Define) — 약 20분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 규정 QA 봇 구축을 바탕으로, 지금부터는 **법령이라는 더 엄격한 문서 유형**에 맞는 규칙을 스스로 정의합니다.

!!! abstract "과제"
    대상 법령과 질문 유형을 확정하고, 원문 인용·해석·주의사항으로 구성되는 답변 형식과 법률 자문 고지 문구를 정리하세요.

**요구사항 정리**

- [ ] 대상 법령·조례 1~2건이 선정되었다 (예: 개인정보보호법, 우리 지자체 특정 조례)
- [ ] 질문 유형 3종이 분류되었다 (용어 해석 / 조문 적용 / 사례 질의)
- [ ] 답변 형식이 정의되었다 (원문 인용 → 평이한 해석 → 주의사항/예외)
- [ ] 법률 자문이 아님을 명시하는 고지 문구가 작성되었다

**활용 도구 & 데이터**

- Dify Knowledge + Chatbot
- 대상 법령 원문 PDF/텍스트 (국가법령정보센터 등에서 확보)
- ChatGPT: 평이한 해석 톤 설계 보조, 질문 유형 브레인스토밍

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    "우리 부서 업무와 가장 직접적으로 닿는 법령 1개"만 먼저 고르세요. 범위가 좁아야 해석 품질이 보장됩니다. 예:

    - 개인정보 관련 민원이 많음 → **개인정보보호법**
    - 시민 생활 조례가 많음 → **○○시 ○○ 조례**

    대상 법령 안에서도 전부 넣지 말고 실제 자주 적용되는 **3~5개 조문**으로 한정해 시작하세요.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    법령 해석 답변은 3단 구조로 고정하면 품질이 안정됩니다:

    ```
    [원문] 제N조(제목) ... 해당 조문 직접 인용
    [해석] 일반 공무원이 이해할 수 있는 평이한 풀이 (3~5줄)
    [주의] 예외/단서/다른 조문 참조 (있다면)
    ※ 본 응답은 법률 자문이 아닙니다. 법적 판단은 소관 부서 또는 법무 담당에게 확인하세요.
    ```

    이 골격을 요구사항에 그대로 적어두면 Phase 2 프롬프트 작성이 빨라집니다.

## Phase 2. 구현하기 (Build) — 약 55분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    법령 원문을 **조문 단위로 청킹**한 지식베이스를 만들고, 챗봇 시스템 프롬프트에 답변 형식과 고지 문구를 반영한 뒤, 9건 테스트로 근거 조항이 올바르게 인용되는지 검증하세요.

**마일스톤**

☐ 법령 원문을 지식베이스에 업로드하고 조문 단위(제N조)로 청킹되도록 설정했다
☐ Chatbot 앱을 생성하고 지식베이스를 연결, Top-K와 score 임계값을 설정했다
☐ 시스템 프롬프트에 "일반 공무원 톤"의 해석 + 법률 자문 아님 고지를 배치했다
☐ 질문 유형 3종에 대해 각 3건, 총 9건을 테스트하고 근거 조문을 표로 기록했다
☐ 답변에 원문 인용과 해석이 모두 포함되는지 확인했다 (둘 중 하나만 나오면 재검증)

**활용 도구 & 데이터**

- Dify Knowledge: Custom Segmentation(조문 단위) 또는 chunk size 조정
- Dify Chatbot: System Prompt, Context 연결
- ChatGPT: 난해한 조문을 평이하게 푸는 문장 리라이팅 보조

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    법령 문서의 관건은 **"조문이 잘리지 않게" 청킹하는 것**입니다. 접근법:

    1. 먼저 기본 chunk size(500자)로 업로드하고 Retrieval Test로 "제N조"가 온전히 뽑히는지 확인
    2. 잘려나가면 chunk size를 800~1200자로 늘리고 overlap을 200자로 조정
    3. 그래도 안 되면 수동으로 조문 단위 텍스트를 분리해 업로드 (조문 하나 = 문서 하나)

    작은 단위부터 시도해 품질이 보장될 때까지 점진적으로 조정하세요.

??? tip "힌트 2: 도구 활용이 막힌다면"
    "평이한 해석" 톤이 안 잡히면 시스템 프롬프트에 샘플 1쌍을 Few-shot으로 넣으세요:

    ```
    예시 입력: "개인정보 '처리'의 정의는 무엇인가요?"
    예시 답:
    [원문] 제2조 2. "처리"란 개인정보의 수집, 생성, 연계, 연동, 기록, 저장 ... 등을 말한다.
    [해석] 쉽게 말해 개인정보를 '다루는 모든 행위'입니다.
      종이에 적어 보관하거나, 시스템에 입력·수정·전달하는 것 모두 해당합니다.
    [주의] 단순히 '보관만' 하는 것도 처리에 포함되므로 목적 달성 후에는 파기해야 합니다.
    ※ 본 응답은 법률 자문이 아닙니다.
    ```

    Few-shot 1~2개만으로도 톤이 크게 안정됩니다.

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    테스트 9건을 질문 유형별로 다음처럼 구성하면 검증이 쉬워집니다:

    | 유형 | 예시 질문 | 기대 근거 |
    |------|----------|-----------|
    | 용어 해석 | "개인정보 처리 위탁이란?" | 제26조 |
    | 조문 적용 | "직원 사진을 SNS에 올려도 되나요?" | 제15조 + 제22조 |
    | 사례 질의 | "민원인 전화번호를 다른 부서에 공유해도 되나요?" | 제17조 |

    각 질문의 **기대 근거 조항**을 미리 정해두면 나중에 채점이 명확합니다.

## Phase 3. 확장하기 (Extend) — 약 25분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    답변이 잘못된 조문을 인용했거나, 해석이 오해 소지가 있는 사례 2건을 찾아 청킹·검색 설정 또는 프롬프트 중 가장 효과적인 지점을 수정하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    법령 해석 오류는 대개 다음 패턴입니다:

    - **이웃 조문 혼동**: 제15조와 제17조를 섞어 답함 → 청킹 단위를 조문 단위로 더 명확히
    - **단서 누락**: "단, ~의 경우에는" 예외 조항을 빠뜨림 → Top-K 증가, 또는 "예외/단서도 반드시 포함" 프롬프트
    - **과해석**: 원문에 없는 결론을 덧붙임 → 프롬프트에 "원문에 없는 결론은 쓰지 말 것" 강조

    한 번에 한 가지만 고치고 같은 9건을 돌려 비교하세요.

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    작동하는 챗봇에 다음 중 하나를 추가해보세요: (1) 관련 판례 요약 PDF 2~3건을 지식베이스에 추가, (2) "민원인 설명용"과 "담당자용" 톤 분기 변수 추가, (3) 다른 법령 1개를 추가 업로드해 법령 선택 변수 구성.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    가장 체감 효과가 큰 확장은 **톤 분기**입니다. 같은 조문이라도:

    - 담당자용 → 조문 번호와 한자어를 유지한 해석
    - 민원인용 → 조문 번호는 참고로만, 쉬운 말로 풀어쓴 안내 + "자세한 사항은 담당 부서에 문의"

    입력 변수 `대상`을 만들고 프롬프트의 해석 단계에서 분기하면 쉽게 구현됩니다.
