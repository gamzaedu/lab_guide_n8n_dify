### 시나리오

> 당신은 시민 만족도 설문 혹은 내부 업무 만족도 설문을 운영하는 담당자입니다.
> 응답이 들어올 때마다 몰라서 중요한 불만을 놓치고, 하루 지나고 나서야 Sheets를 열어 직접 세어 보고하는 상황입니다.
> 응답이 들어올 때마다 Slack 채널에 즉시 알림이 가고, 매일 저녁 6시에 당일 집계(응답 수·주요 의견·낮은 점수 항목)를 Slack과 Sheets 대시보드에 정리해 자동 공유하는 워크플로우를 만드세요.

### 최종 산출물

- 작동하는 n8n 워크플로우 2개 (실시간 알림용 / 일일 집계용)
- Google Sheets 대시보드(일자·응답수·평균·낮은점수 항목)
- Slack 일일 리포트 메시지 샘플

## Phase 1. 정의하기 (Define) — 약 30분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 Schedule 트리거와 Merge 노드를 바탕으로, 지금부터는 "실시간 경로와 배치 경로를 어떻게 나눌지"를 스스로 설계합니다.

!!! abstract "과제"
    설문 문항을 보고 (1) 실시간으로 알려야 할 응답의 조건과 (2) 일일 리포트에 담을 지표 3~5개를 정의하세요. 두 경로가 서로 어떻게 분리되는지 한 장 다이어그램으로 그려봅니다.

**요구사항 정리**

- [ ] 설문 문항과 척도(1~5점 등)가 정리되었다
- [ ] 즉시 알림 기준이 정해졌다 (예: "점수 2 이하" 또는 "서술형에 특정 키워드")
- [ ] 일일 리포트 지표 3~5개가 정해졌다 (응답 수, 평균, 최저 점수 항목, 대표 코멘트 등)
- [ ] 배포 채널 2종이 정해졌다 (실시간용 Slack 채널 / 일일 리포트용 채널)
- [ ] 개인정보 처리 규칙이 정해졌다 (이름·직위 노출 여부)

**활용 도구 & 데이터**

- Google Form (5~7문항, 리커트 + 서술형 1개 포함)
- Google Sheets (Form 응답 자동 저장 + 대시보드 탭 분리)
- ChatGPT: 지표 아이디어, 프롬프트 초안

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    "이 설문이 없었더라면 어떤 상황이 문제였을까?"를 역으로 질문하세요.
    그 '피했으면 하는 상황'이 실시간 알림 조건의 단서입니다. (예: 심각한 불만이 상급자에게 늦게 보고되는 상황)

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에게 다음과 같이 물어보세요:
    "공공기관 내부 만족도 설문 기준 '실시간 경보 조건'과 '일일 집계 지표'를 각각 3~5개씩 제안해줘. 조건은 수치나 키워드 기반으로 구체적으로."

## Phase 2. 구현하기 (Build) — 약 80분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    두 개의 워크플로우를 만드세요. (A) 응답이 들어올 때마다 조건을 검사해 즉시 Slack 알림, (B) 매일 저녁 6시 Schedule로 당일 응답을 집계해 Sheets 대시보드와 Slack 리포트에 기록.

**마일스톤**

☐ Google Form 응답이 Sheets에 자동 저장되는 것을 확인했다
☐ (A) Webhook 또는 Sheets Trigger로 신규 응답이 n8n에 들어오는 것을 확인했다
☐ (A) IF 노드로 실시간 알림 조건이 동작하고 Slack 메시지가 전송됐다
☐ (B) Schedule Trigger로 18:00 실행이 설정됐다
☐ (B) Sheets에서 "오늘 날짜" 행만 필터링해 집계했다 (Code 또는 Expression)
☐ (B) 집계 결과가 대시보드 탭에 한 행으로 추가되고, Slack에 요약 메시지가 전송됐다

**활용 도구 & 데이터**

- n8n: Webhook(or Sheets Trigger) / IF / Schedule / Google Sheets / Merge / Slack
- Google Form & Sheets 연결 유지
- ChatGPT: 집계 로직, Slack 메시지 포맷 설계

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    "한 경로씩 완성"이 핵심입니다.

    1. 먼저 (A) 실시간 알림만 완성: 테스트 응답 1건 → Slack 도착 확인
    2. 그 다음 (B) 일일 집계 워크플로우 별도로 완성
    3. 두 워크플로우가 같은 Sheet를 건드리니, 탭을 분리해서 충돌을 피합니다 (`raw`, `dashboard`)

??? tip "힌트 2: 도구 활용이 막힌다면"
    "오늘 응답만" 필터링하는 Expression:
    ```
    {{ $json.timestamp.startsWith($now.toFormat('yyyy-MM-dd')) }}
    ```
    또는 Sheets 노드의 Filter에 `날짜 = TODAY()` 수식을 세워도 됩니다.

    Merge 노드는 "서술형 응답 수집 결과"와 "점수 집계 결과"를 합칠 때 유용합니다. 같은 timestamp 기준으로 Merge.

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    Slack 실시간 알림 포맷:
    ```
    :warning: *저점 응답 포착* (점수 {{score}})
    • 문항: {{item}}
    • 코멘트: {{comment}}
    • 응답 시각: {{ts}}
    ```

    Slack 일일 리포트 포맷:
    ```
    :bar_chart: *오늘의 설문 리포트 ({{date}})*
    • 응답 수: {{count}}
    • 평균 점수: {{avg}}
    • 최저 점수 항목: {{worst_item}} ({{worst_avg}})
    • 대표 코멘트: "{{top_comment}}"
    ```

## Phase 3. 확장하기 (Extend) — 약 40분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    테스트 응답 10건(저점/고점/이상치 포함)을 넣어 보고, 즉시 알림의 오경보·누락 사례를 2건 이상 찾아 조건을 개선하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    다음 두 가지를 분리해 살피세요.

    - 오경보(울리지 않아도 됐는데 울림) → 조건에 AND/예외 추가
    - 누락(울려야 했는데 안 울림) → 조건에 OR 추가 또는 키워드 보강

    서술형 응답에 대해서는 OpenAI로 감성 분석을 덧붙여 "점수는 보통인데 글에 불만이 강함" 케이스도 잡을 수 있습니다.

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    다음 중 하나를 추가하세요. (1) 주간 리포트 워크플로우 추가 (일일을 Merge로 합쳐 주간 트렌드 생성), (2) 저점 응답자에게 자동 회신(감사 + 후속 조치 안내) 발송, (3) 대시보드 그래프용 데이터 포맷을 만들어 Looker Studio에 연결.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    "실시간 + 배치" 구조는 VOC·콜센터 로그·근무일지 등 거의 모든 운영 데이터에 적용됩니다.
    한 경로는 문제를 빨리 잡고, 다른 경로는 큰 흐름을 본다는 차이를 기억하세요.
