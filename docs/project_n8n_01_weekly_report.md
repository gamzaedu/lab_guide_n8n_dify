# Project 01. 주간 업무 리포트 자동 생성·발송

| 항목 | 내용 |
|------|------|

| 소요 시간 | 2교시 (100분) |
| 선행 Lab | **Lab 02. Trigger 노드 비교**, **Lab 08. Expression 마스터**, **Lab 18. Google Sheets**, **Lab 19. Gmail 노드**, **Lab 21. OpenAI 노드** |
| 사용 도구 | n8n, Google Sheets, ChatGPT(OpenAI API), Gmail |

### 시나리오

> 당신은 주무관으로서 팀장에게 매주 월요일 오전 "지난주 업무 현황"을 보고해야 합니다.
> 지금까지는 금요일 오후에 Google Sheets를 열어 건수를 세고, 워드에 정리한 뒤 월요일 아침 메일로 보냈습니다. 본 업무에 방해가 되고 실수도 잦습니다.
> 이 과정을 n8n으로 자동화해, 매주 월요일 아침 9시에 팀장 메일함에 리포트 초안이 도착하도록 만드세요. 제공된 샘플 시트를 사용하며, 팀장이 5분 안에 전체 상황을 파악할 수 있어야 합니다.

### 최종 산출물

- 작동하는 n8n 워크플로우 (Schedule → Sheets → GPT 요약 → Gmail 발송)
- 실제 발송된 샘플 리포트 메일 1통
- 워크플로우 설계 메모 (노드 흐름 + 판단 근거)

## Phase 1. 정의하기 (Define) — 약 20분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 Schedule 트리거 비교와 Sheets 데이터 읽기를 바탕으로, 지금부터는 스스로 문제를 정의합니다.

!!! abstract "과제"
    팀장이 "주간 리포트에서 꼭 보고 싶어하는 것" 3~5가지를 정의하고, 각 항목에 필요한 Sheets 컬럼과 계산 규칙을 정리하세요.

**요구사항 정리**

- [ ] 리포트의 독자(팀장)와 발송 시점(요일·시간)이 정의되었다
- [ ] 핵심 지표 3개 이상이 작성되었다 (예: 처리율, 지연 건수, 다음 주 주요 일정)
- [ ] 각 지표에 필요한 Sheets 컬럼이 연결되었다
- [ ] 메일 제목·본문의 톤(보고체/요약체)이 결정되었다

**활용 도구 & 데이터**

- 제공 샘플 시트: `Input/weekly_tasks.xlsx` (Google Sheets로 업로드하여 사용)
- ChatGPT: 지표 아이디어 브레인스토밍, 리포트 문체 제안

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    "팀장이 이 메일을 열었을 때 가장 먼저 눈으로 찾을 3줄은 무엇일까?"를 출발점으로 삼아보세요.
    예: "이번 주 처리 몇 건?", "지연/문제 있는 건?", "다음 주 뭐가 중요한가?"

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에게 다음과 같이 물어보세요:
    "나는 공공기관 주무관이고 매주 월요일 팀장에게 지난주 업무 현황을 보고해야 합니다. 5분 안에 읽히는 주간 리포트에 꼭 들어가야 할 핵심 항목 5가지와 그 이유를 표로 정리해주세요."

    AI의 제안을 그대로 쓰지 말고, 우리 팀 상황에 맞게 3~5개로 추려 직접 정의하세요.

## Phase 2. 구현하기 (Build) — 약 55분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    Phase 1에서 정의한 지표를 포함하는 주간 리포트 메일을 매주 월요일 오전 9시에 자동 발송하는 n8n 워크플로우를 만드세요. Schedule 트리거 → Sheets 읽기 → Expression으로 지표 계산 → GPT로 요약 문장 생성 → Gmail 발송 순서로 구성합니다.

**마일스톤**

☐ Schedule Trigger 노드를 추가하고 "매주 월요일 09:00" 스케줄이 올바르게 설정됐다
☐ Google Sheets 노드로 샘플 시트를 읽어 출력 데이터를 눈으로 확인했다
☐ Edit Fields / Expression으로 핵심 지표 1개 이상(예: 완료율, 지연 건수)을 계산했다
☐ OpenAI 노드가 지표를 받아 팀장용 요약 3줄을 생성했다
☐ Gmail 노드로 본인 메일함에 테스트 발송했고, 제목·본문이 의도대로 들어갔다

**활용 도구 & 데이터**

- n8n: Schedule Trigger / Google Sheets / Edit Fields / OpenAI / Gmail 노드
- Google Sheets: 샘플 시트 업로드 후 공유 범위·권한 확인
- ChatGPT: 프롬프트 문장 다듬기, Expression 문법 질문

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    한 번에 완성하려 하지 말고 아래 순서로 쌓으세요.

    1. Manual Trigger로 먼저 시작 → Sheets 노드만 붙여 데이터 읽기 성공
    2. Edit Fields로 필요한 컬럼만 남기기
    3. Expression으로 지표 1개 계산 → 출력 확인
    4. OpenAI 노드 추가 → 프롬프트에 지표 값 삽입해 요약 받기
    5. Gmail 노드로 본인에게 테스트 발송
    6. **마지막에** Manual을 Schedule Trigger로 교체

    각 단계마다 Execute를 눌러 중간 결과를 눈으로 확인하세요.

??? tip "힌트 2: 도구 활용이 막힌다면"
    Expression 예: `{{ $json.status === '완료' ? 1 : 0 }}`를 항목마다 적용하고 합계를 내면 완료 건수를 얻을 수 있습니다.

    OpenAI 프롬프트 예시 골격:
    ```
    너는 공공기관 팀장을 위한 주간 보고 작성자다.
    아래 지표를 보고, 5문장 이내로 핵심 요약 + 주의할 점 + 다음 주 포인트 순서로 작성해라.
    톤은 간결한 보고체. 이모지 사용 금지.

    지표:
    - 처리 건수: {{ $json.completed }}
    - 지연 건수: {{ $json.delayed }}
    - 다음 주 주요 일정: {{ $json.next_week }}
    ```

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    Gmail 본문 최종 구조 예시:

    ```
    [주간 업무 리포트] 2026-04-20 (월)

    ■ 핵심 요약 (GPT 생성)
    지난주 총 27건 중 23건을 완료(처리율 85%), 지연 4건은 A부서 협조 대기. 
    다음 주는 예산 마감과 민원 회의가 동시에 몰리므로 월·화 집중 처리가 필요합니다.

    ■ 지표 상세
    - 완료: 23건 / 진행: 4건 / 지연: 4건
    - 완료율: 85% (전주 대비 +3%p)

    ■ 다음 주 주요 일정
    - 4/22 예산 집행 마감
    - 4/24 민원 대응 점검 회의
    ```

## Phase 3. 확장하기 (Extend) — 약 25분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    완성된 리포트 메일을 팀장 입장에서 읽어보고, 개선할 점 2가지를 찾아 프롬프트 또는 Expression을 수정하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    다음 기준으로 점검하세요.

    - 핵심 3줄이 스크롤 없이 보이는가?
    - 숫자만 있고 "그래서 어떻다"는 해석이 빠지지 않았는가?
    - 다음 주 행동을 결정할 수 있는 정보가 포함됐는가?

    결과 메일을 ChatGPT에 붙여넣고 "팀장이 이 메일을 받았을 때 불명확하거나 빠진 정보가 있다면 짚어줘"라고 피드백을 받는 것도 좋습니다.

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    작동하는 워크플로우에 다음 중 하나를 추가해보세요. (1) 지연 건수가 임계치를 넘으면 IF 분기로 상급자에게 별도 경고 메일 발송, (2) 같은 구조로 "일일 민원 현황" 리포트 버전 복제, (3) Gmail 대신 Slack DM으로 동시 발송.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    주기와 독자가 바뀌면 리포트가 바뀝니다.

    - 팀장 + 주간 → 진행·리스크 중심
    - 본인 + 일간 → 오늘 할 일 요약
    - 과장/국장 + 월간 → 성과·이슈 중심

    가장 쉬운 확장은 기존 워크플로우를 복제(Duplicate)해 스케줄·시트·프롬프트만 바꾸는 것입니다.
