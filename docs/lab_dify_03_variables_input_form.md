# Lab 3. 변수(Variables)로 입력 폼 만들기

## 개요

| 항목 | 내용 |
|------|------|
| 학습 목표 | Text/Paragraph/Select 변수를 조합해 입력 폼을 만들고, 프롬프트에 변수를 삽입해 재사용 가능한 업무 보조기를 만들 수 있다 |
| 핵심 기능 | Variables, Input Form, `{{변수명}}` 삽입 |
| 소요 시간 | 약 40분 |
| 사전 준비 | Lab 02 완료 (System Prompt 구조화 경험) |
| 난이도 | ★☆☆☆☆ |

### Step 1. 탐색하기 (Exploring)

??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"
        남이 알려준 정답보다 나의 시도가 더 가치 있습니다.

    **[의미]**

    수업에서 배운 내용을 직접 적용해보는 단계입니다.

    **[지침]**

    - 가능한 직접 손으로 타이핑해보고 직접 생각해봅니다.
        - 5분 정도는 AI 에서 손을 떼고 펜을 들어보세요!
    - 한번의 완벽한 결과보다 다양한 시도를 통한 점진적 개선을 추구합니다.
    - 실패를 두려워하지 마세요. 빠른 실패는 오히려 빠른 성장을 가져옵니다!

!!! abstract "과제"
    Dify Chatbot 앱에 **Text / Paragraph / Select 변수**를 각 1개씩 추가해 **민원 요약기** 를 만드세요. 입력 폼이 채팅창 위에 자동으로 나타나는 것을 확인한 뒤, 실제 민원 1건을 입력해 요약 결과를 확인합니다.

**활용 도구 & 데이터**

- **도구:** Dify Chatbot (앱 이름: `Lab 03 - 민원 요약기`)
- **추가할 변수 3종:**

| 변수명 | 타입 | 라벨 | 설명 |
|--------|------|------|------|
| applicant_name | Text | 민원인 이름 | 짧은 단문 입력 |
| complaint_body | Paragraph | 민원 내용 | 여러 줄 입력 |
| urgency | Select | 긴급도 | 옵션: `일반`, `중요`, `긴급` |

- **System Prompt 템플릿:**

```
당신은 민원 요약 전문가입니다.
아래 민원을 3줄 이내로 요약하고, 핵심 키워드 3개를 추출하세요.

민원인: {{applicant_name}}
긴급도: {{urgency}}

민원 내용:
{{complaint_body}}

출력 포맷:
1) 요약: (3줄 이내)
2) 핵심 키워드: #키워드1 #키워드2 #키워드3
3) 추천 처리 순서: (긴급도에 맞춰 1~3 순위 제안)
```

**추가 순서:**

1. Chatbot 앱 생성 → 이름 `Lab 03 - 민원 요약기`
2. 좌측 하단 **Variables** 영역 → **[+ Add]** 클릭
3. Type: `Text Input` → Variable Key: `applicant_name` / Label: `민원인 이름`
4. 다시 **[+ Add]** → Type: `Paragraph` → Key: `complaint_body` / Label: `민원 내용`
5. 다시 **[+ Add]** → Type: `Select` → Key: `urgency` / Label: `긴급도` / Options: `일반, 중요, 긴급`
6. Instructions에 위 System Prompt 복사, `{{변수명}}` 형태로 삽입되어 있는지 확인
7. 우측 Debug & Preview → 입력 폼이 채팅 위에 나타남 → 값 입력 후 질문창에 아무 문자나 입력(예: `요약해줘`) → 결과 확인

??? note "예시"

    > **입력 폼 값**
    >
    > | 변수 | 입력값 |
    > |------|--------|
    > | 민원인 이름 | 김민수 |
    > | 민원 내용 | 아파트 단지 내 불법주차 차량이 3일째 소방도로를 막고 있습니다. 112 신고했는데 구청 담당이라고 해서 왔습니다. |
    > | 긴급도 | 긴급 |
    >
    > **챗봇 응답:**
    >
    > ```
    > 1) 요약:
    >   - 아파트 소방도로에 불법주차 차량이 3일째 방치
    >   - 112 신고 후 구청 담당으로 안내받음
    >   - 긴급 조치 필요 (소방도로 확보)
    >
    > 2) 핵심 키워드: #불법주차 #소방도로 #긴급견인
    >
    > 3) 추천 처리 순서:
    >   1순위: 현장 출동 + 과태료 부과 및 견인 조치
    >   2순위: 아파트 관리사무소 통보
    >   3순위: 민원인 처리 결과 통지
    > ```
    >
    > **확인 포인트:** (1) 입력 폼이 채팅창 위에 표시됐는지, (2) `{{urgency}}` 가 "긴급"으로 치환돼 긴급 처리 순서가 제안됐는지, (3) 3개의 출력 항목이 모두 등장했는지.

### Step 2. 분석과 성찰 (Analyzing & Reflecting)

??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"
        객관적인 피드백을 구하되, 판단은 스스로 합니다.

    **[의미]**

    Step 1 에서 만든 결과물을 한 발 물러서서 객관적으로 바라보는 단계입니다.
    혼자서는 보이지 않던 것들이, 다른 시선을 통해 드러납니다.

    **[지침]**

    - 1단계 작업물에 대해, 객관적인 피드백을 얻습니다
    - 다른 탐색하기 결과물을 확인합니다
        - 다른 동료의 작업 결과물 | AI 의 생성 결과물 | 웹검색 등
    - AI 를 사용할 땐 반드시 비판적인 자세를 유지합니다.
        - AI 가 생성한 결과물 및 지침은 그대로 받아들이지 않고, 의심합니다.

!!! abstract "과제"
    **변수를 쓰지 않는 버전(A)** 과 **변수를 쓰는 버전(B)** 을 같은 민원 3건으로 실행해, 입력 편의성·출력 일관성·재사용성을 비교 정리하세요.

**활용 도구 & 데이터**

- **도구:** Step 1 앱을 복제해 A/B 두 버전 운영
- **비교 설계:**

| 버전 | 입력 방식 |
|------|-----------|
| A (변수 없음) | 채팅창에 이름/내용/긴급도를 자유롭게 한 번에 입력 |
| B (변수 3종) | Step 1 그대로 |

- **테스트 민원 3건:** 불법주차 / 가로등 고장 / 도로 침수

**측정 항목:**

| 항목 | 측정 기준 |
|------|----------|
| 입력 편의성 | 민원 담당자가 누락 없이 입력하기 쉬운가 (O/△/X) |
| 출력 일관성 | 3건의 출력 포맷이 동일한가 (O/△/X) |
| 재사용성 | 다른 사람에게 전달했을 때 바로 쓸 수 있는가 (O/△/X) |

**비교 방법:**

1. A 앱: 변수 3개를 모두 삭제하고 Instructions의 `{{변수명}}` 도 평문으로 바꿈
2. B 앱: Step 1 그대로
3. 3건 민원을 각각 A/B에 입력 → 출력 비교

**예시**

> **비교 분석표 — A(변수 없음) vs B(변수 3종)**
>
> | 항목 | A 결과 | B 결과 | 승자 |
> |------|--------|--------|------|
> | 불법주차 — 입력 편의성 | 담당자가 문단으로 써야 해 누락 가능 (△) | 폼 3칸만 채우면 완료 (✅) | B |
> | 가로등 고장 — 출력 일관성 | 긴급도 누락 시 3순위 제안도 빠짐 (△) | 긴급도 필수이므로 항상 포함 (✅) | B |
> | 도로 침수 — 재사용성 | 프롬프트 이해가 필요해 다른 사람이 쓰기 어려움 (X) | 폼만 보고 바로 사용 가능 (✅) | B |
>
> **분석 결과:**
>
> - 변수를 쓰면 **필수 입력값을 강제할 수 있어** 누락 민원이 줄어듦 (Select의 효과가 가장 큼)
> - 실무에서 "프롬프트를 모르는 동료"도 쓸 수 있게 하려면 변수화가 필수
> - 단, 자유 형식 입력이 중요한 창작 업무에서는 변수가 오히려 제약이 될 수 있음 — 업무 성격에 맞게 선택

### Step 3. 창조하기 (Creating)

??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"
        완성품이 아니라, 작동하는 시작품을 만듭니다.

    **[의미]**

    앞서 얻은 통찰을 바탕으로, 나만의 결과물을 만들어 봅니다.

    **[지침]**

    - 2단계에서 얻은 피드백을 원하는 만큼 반영합니다.
        - 중요한 점은, 받아들일 수 있는 만큼만 받아들이는 것입니다.
    - 내 상황과 스타일에 맞게 최적화된 결과물을 만들어 봅니다.
    - 처음부터 완성하는 것이 아닌 계속해서 개선해 나가는 것입니다.
        - 가장 작은 단위부터 프로토타입으로 만들어 보세요

!!! abstract "과제"
    **본인 업무의 반복 입력 양식**(예: 민원 분류, 회의록 작성, 보도자료 초안) 중 하나를 골라 **변수 4개 이상**을 사용한 입력 폼 챗봇을 완성하세요. 최소 1개의 Select 변수를 포함하고, Required(필수) 설정을 걸어 누락 입력을 방지합니다.

**활용 도구 & 데이터**

- **도구:** Dify Chatbot
- **설계 포인트:**

| 항목 | 설명 |
|------|------|
| 업무 선택 | 본인 업무 중 "반복 입력"이 있는 1가지 |
| 변수 개수 | 4개 이상 |
| 변수 구성 | Text 1+, Paragraph 1+, Select 1+ 이상 |
| Required 설정 | 핵심 변수 2개 이상에 필수 체크 |

**완성 순서:**

1. 업무 선택 후 필요한 입력 항목을 종이에 먼저 적기 (예: 제목, 요청부서, 기한, 상세내용)
2. 각 항목을 어떤 변수 타입으로 만들지 결정
3. Dify 앱에서 변수 추가 + Required 체크 + Options 설정(Select)
4. System Prompt에 `{{변수명}}` 삽입하고 출력 포맷 정의
5. 3건 테스트 → 폼 입력 → 출력 확인 → Publish

**예시**

> **완성 챗봇: "부서 협조요청 공문 초안 작성기"**
>
> **변수 4종:**
>
> | 변수명 | 타입 | 라벨 | Required |
> |--------|------|------|----------|
> | title | Text | 공문 제목 | ✅ |
> | to_dept | Select | 수신 부서 | ✅ (옵션: 기획과 / 재정과 / 총무과 / 홍보실) |
> | deadline | Text | 회신 기한 (YYYY-MM-DD) | ✅ |
> | body | Paragraph | 요청 본문 | ✅ |
>
> **System Prompt (발췌):**
>
> ```
> 당신은 공공기관 공문 작성 보조자입니다. 아래 정보를 바탕으로
> 공식적인 협조요청 공문 초안을 생성하세요.
>
> 제목: {{title}}
> 수신: {{to_dept}}
> 회신 기한: {{deadline}}
> 본문 핵심: {{body}}
>
> 출력 포맷:
> [수신] ...
> [제목] ...
> [본문] (3단락: 배경 / 요청사항 / 회신 방법)
> [협조 요청 기한] ...
> [담당자] ... (빈 칸으로 남김)
> ```
>
> **출력 샘플:**
>
> ```
> [수신] 기획과장
> [제목] 공무원 AI 교육 운영 계획 협조 요청
> [본문]
> 1. 관련하여 2026년도 공무원 대상 AI 교육을 4월 ~ 5월 중 운영할 예정입니다.
> 2. 이에 아래와 같이 교육장 배정 및 예산 집행 관련 협조를 요청드립니다.
>   - 교육장: 본관 3층 대회의실 (5일, 오전 9시~오후 5시)
>   - 예산: 강사료 및 교재비 지원
> 3. 회신은 아래 기한 내 공문 또는 메일로 부탁드립니다.
> [협조 요청 기한] 2026-04-30
> [담당자] (기입)
> ```

!!! tip ""
    결과가 만족스럽지 않다면 Step 1이나 Step 2로 돌아가 반복할 수 있습니다.

**예시**

> - **변수를 System Prompt 와 Opening Statement 에 모두 쓰면 어떤 효과가 있을까?**
>     - `안녕하세요 {{applicant_name}}님!` 처럼 Opening Statement에도 변수를 넣어보세요. 사용자 몰입도가 달라지는지 비교해보세요.
> - **같은 업무를 "변수 기반 입력 폼" vs "순차 질문 Chatflow"로 만들면 어느 쪽이 더 좋을까?**
>     - Lab 11(Chatflow)까지 학습한 뒤 두 방식을 같은 업무로 만들어 비교해보세요. 어느 쪽이 누락이 적고, 어느 쪽이 사용자 편의가 좋은지 근거를 정리해보세요.
