# Project 02. 공문 톤앤매너 변환기

## 프로젝트 개요

| 항목 | 내용 |
|------|------|

| 소요 시간 | 1교시 (50분) |
| 선행 Lab | **Lab 02 · 프롬프트 템플릿 민원 답변**, **Lab 03 · 변수로 입력 폼 만들기** |

### 시나리오

> 당신은 신규 공무원입니다.
> 회의에서 구두로 논의된 사항이나 메모장에 적어둔 결정 사항을 공문 형태로 올려야 하는데, 경어체·종결어미·'통보/협조/보고' 형식이 매번 헷갈려 한 건 쓰는 데 30분씩 걸립니다. 당신은 구어체 원문과 문서 종류·수신기관만 입력하면 공문체 초안으로 변환해주는 사내 변환기를 만들려 합니다.

### 최종 산출물

- 구어체/메모 원문 → 공문체 변환 Dify Chatbot 앱
- Few-shot 예시 3~5쌍이 반영된 시스템 프롬프트
- 실제 업무 메모 1건 이상으로 검증된 변환 결과

## Phase 1. 정의하기 (Define) — 약 10분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 프롬프트 템플릿과 변수 입력 폼을 바탕으로, 지금부터는 스스로 문제를 정의합니다.

!!! abstract "과제"
    공문 톤앤매너 규칙과 문서 종류별 관용구, 그리고 프롬프트에 넣을 Before/After 예시를 정리하세요.

**요구사항 정리**

- [ ] 공문 톤앤매너 규칙 5개가 정리되었다 (경어체 · 종결어미 · 한자어 활용 · 존칭 · 문장 길이)
- [ ] 문서 종류별 도입·맺음말 관용구가 수집되었다 (통보 · 협조 · 보고 각 1세트 이상)
- [ ] Before/After 샘플 3~5쌍이 작성되었다 (Few-shot 재료)
- [ ] 변환 거부 조건이 정의되었다 (너무 짧거나 맥락이 부족한 입력 처리 규칙)

**활용 도구 & 데이터**

- Dify Chatbot 앱 (Cloud)
- ChatGPT: 문서 종류별 관용구 수집, 규칙 정리 보조
- 참고 자료: 최근 내 기관 공문 2~3건 (스타일 참고)

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    "내가 실제로 썼던 메모"와 "그 메모로 최종 보냈던 공문"을 나란히 펼쳐놓으세요. 두 문장을 비교하면서 바뀐 지점만 체크하면, 그 자체가 규칙 목록이 됩니다.

    예: "확인해주세요" → "확인하여 주시기 바랍니다" / "알려드립니다" → "통보드리오니 업무에 참고하시기 바랍니다".

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에게 이렇게 물어보세요:
    "한국 공공기관 공문의 톤앤매너를 '통보/협조/보고' 세 가지 문서 종류별로 도입부 관용구 2개와 맺음말 관용구 2개씩 알려주세요. 그리고 공문에서 피해야 할 구어적 표현 5가지도 나열해주세요."

    AI 결과는 반드시 내 기관 최근 공문과 대조해 실제로 사용하는 표현으로만 추려내세요.

## Phase 2. 구현하기 (Build) — 약 28분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    Phase 1에서 정리한 규칙과 Few-shot 예시를 Dify Chatbot 앱의 시스템 프롬프트에 구성하고, 5개 샘플로 변환 품질을 검증하세요.

**마일스톤**

☐ Dify에서 새 Chatbot 앱을 생성하고 변수 3개(원문 · 문서종류 · 수신기관)를 등록했다
☐ 시스템 프롬프트에 톤앤매너 규칙 + Few-shot 예시 3쌍을 배치했다
☐ 변환 거부 시 안내 문구 규칙을 프롬프트에 추가했다
☐ 서로 다른 문서 종류(통보 · 협조 · 보고)로 5개 샘플을 변환해 경어/종결어미를 검증했다
☐ 실제 업무 메모 1건을 넣어 동료에게 공문으로 쓸 만한지 피드백을 받았다

**활용 도구 & 데이터**

- Dify Chatbot: Variables, System Prompt, Preview
- ChatGPT: Few-shot 예시 리팩터링, 프롬프트 다듬기
- 샘플 메모 5건 (회의 메모 2 · 결정 사항 2 · 요청 메모 1)

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    Few-shot 학습은 "규칙 설명 3줄 + 예시 3쌍"이 "규칙 설명 20줄 + 예시 0쌍"보다 보통 효과가 큽니다. 먼저 Before/After 쌍을 3개 확보한 뒤, 규칙 설명은 요약해서 넣으세요.

    변수 3개는 프롬프트 안에 `[원문]{{원문}} [문서종류]{{문서종류}} [수신기관]{{수신기관}}` 형태로 명시적 라벨과 함께 배치하면 모델이 역할을 헷갈리지 않습니다.

??? tip "힌트 2: 도구 활용이 막힌다면"
    "거부 조건"은 Dify에서 별도 노드 없이 프롬프트로 구현할 수 있습니다. 예:
    "원문이 10자 미만이거나 사실 정보가 부족하면, 변환하지 말고 '맥락이 부족합니다. 사업명·일시·요청사항을 추가해 주세요'로 응답할 것."

    Preview에서 짧은 원문을 일부러 넣어 거부 동작이 제대로 나오는지 반드시 확인하세요.

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    Few-shot 쌍 예시:

    ```
    [Before - 메모]
    다음주 월요일에 교육 있으니까 해당 부서 다 참석하라고 알려주세요.

    [After - 통보 공문체]
    다음 주 월요일 예정된 OO 교육과 관련하여 해당 부서 전 직원의 참석이 필요함을
    알려드리오니, 업무에 차질이 없도록 협조하여 주시기 바랍니다.
    ```

    이런 쌍을 3~5개 넣고 마지막에 `[Before]{{원문}} [After]` 를 비워 둬 모델이 그 자리를 채우도록 유도합니다.

## Phase 3. 확장하기 (Extend) — 약 12분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    변환 결과 5건을 점검하여 부자연스러운 결과(과도한 한자어·잘못된 존칭·구어 잔존 등) 2건을 선정하고, Few-shot 예시를 교체하거나 새 쌍을 추가해 다시 테스트하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    많이 틀리는 유형은 보통 정해져 있습니다:

    - **존칭 과다**: "~해주시길 부탁드리오며 감사드리오니" 같이 중복
    - **구어 잔존**: "~라고 생각합니다", "~거든요" 미변환
    - **시제 꼬임**: 과거/미래가 현재로 바뀜

    가장 자주 발생한 유형 하나를 고르고, 그 유형만 교정한 Before/After 쌍을 Few-shot에 추가하세요.

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    작동하는 챗봇에 다음 중 하나를 추가해보세요: (1) "간결 이메일 톤" 분기, (2) "내부 보고용 요약" 분기, (3) "시민 대상 안내문 톤"(평이한 구어·배려 문구) 분기.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    수신자가 바뀌면 톤이 바뀝니다. 문서종류 변수에 값 하나만 추가하면 분기가 늘어납니다:

    - `이메일` → 짧고 쉬운 문장, 존칭 최소
    - `내부 보고` → 경어 유지, 결론 먼저
    - `시민 안내` → 쉬운 말, 구어적 존중

    시스템 프롬프트의 조건 분기에 "if 문서종류 == 이메일 then ..." 형태로 규칙을 추가하세요.
