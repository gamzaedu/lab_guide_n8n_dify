|------|------|

### 시나리오

> 당신은 ○○시 민원실에서 근무하는 주무관입니다.
> 매일 수십 건씩 접수되는 민원을 유형별로 분류하고 담당 부서에 전달하는 업무가 반복되고 있습니다. 단순 분류·답변 작성에 하루의 상당 시간을 쓰고 있어, 실제 까다로운 민원 해결에 집중하기 어렵습니다.
> 접수된 민원을 AI가 자동 분류하고 답변 초안을 생성해주며, 담당 부서에 자동 전달되는 시스템을 Dify Chatflow와 n8n으로 구축해야 합니다.

### 최종 산출물

- 작동하는 Dify Chatflow (민원 분류 + 답변 초안 생성)
- n8n 워크플로우 (Dify 결과를 받아 담당 부서로 자동 라우팅)
- 설계 문서 (분류 카테고리 체계, 부서 매핑 규칙, 오분류 대응 방안)

## Phase 1. 정의하기 (Define) — 약 20분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 IF/ELSE 민원 분류와 Chatflow 설계를 바탕으로, 지금부터는 스스로 문제를 정의합니다.

!!! abstract "과제"
    처리 대상 민원의 **카테고리 7개 이내**를 정의하고, 각 카테고리별 담당 부서 매핑과 답변 초안의 필수 요소를 정리하세요.

**요구사항 정리**

- [ ] 민원 분류 카테고리 체계가 7개 이내로 정의되었다
- [ ] 각 카테고리별 담당 부서 매핑이 작성되었다
- [ ] 답변 초안의 필수 요소(인사말·본문·마무리·연락처)가 명확히 정의되었다
- [ ] 분류 신뢰도가 낮을 때(예: 애매한 민원)의 처리 방식이 정해졌다

**활용 도구 & 데이터**

- `Result/Output/project/project_1_시민문의봇/data/sample_minwon.xlsx` — 샘플 민원 30건
- `Result/Output/project/project_1_시민문의봇/data/categories.xlsx` — 분류 체계 예시 7개
- `Result/Output/project/project_1_시민문의봇/data/departments.xlsx` — 부서 목록 17개와 담당업무
- `Result/Output/project/project_1_시민문의봇/data/edge_cases.xlsx` — 분류 경계 케이스 10건 (개선 루프 참고)
- ChatGPT: 분류 체계 검토 및 답변 톤 조언

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    `sample_minwon.xlsx`를 열어 30건의 민원을 직접 눈으로 훑어보세요. 비슷한 주제끼리 묶다 보면 자연스럽게 5~7개의 카테고리가 보입니다.

    "이 민원은 어느 부서에 가야 할까?"라는 질문으로 묶으면 부서 기준 분류가, "시민이 무엇을 원하는가?"로 묶으면 요청 유형 기준 분류가 만들어집니다. **두 관점 중 하나를 선택**하세요.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에 다음과 같이 물어보세요:

    "나는 공공기관 민원 담당자이고, 민원을 AI로 자동 분류·답변 초안 생성하려 한다. 처음에는 단순하게 시작하려면 몇 개의 카테고리가 적절할까? 그리고 분류가 애매할 때의 대응 전략은?"

    핵심: **처음부터 완벽할 필요 없음**. 5~7개로 시작해 나중에 세분화하는 전략이 더 현실적입니다.

## Phase 2. 구현하기 (Build) — 약 55분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    민원 텍스트를 입력받아 **카테고리 분류 + 답변 초안 생성**을 수행하는 Dify Chatflow를 만들고, n8n으로 분류 결과에 따라 담당 부서에 메일/알림을 보내는 파이프라인을 완성하세요.

**마일스톤**

☐ Dify Chatflow에 시작 노드 + 분류 LLM 노드를 연결해 카테고리가 출력되었다
☐ 답변 템플릿을 참조해 답변 초안 생성 노드가 작동한다
☐ Chatflow 전체가 한 번의 입력으로 `카테고리 + 담당부서 + 답변초안` 세 필드를 반환한다
☐ n8n에서 HTTP Request 노드로 Dify API를 호출해 결과를 받았다
☐ n8n의 분기(IF/Switch)로 카테고리별 담당 부서 이메일로 전송되도록 구성했다

**활용 도구 & 데이터**

- Dify Chatflow: 시작 → LLM(분류) → 변수 저장 → LLM(답변 생성) → 종료
- n8n: Webhook 또는 Schedule 트리거 → HTTP Request(Dify) → Switch → Gmail/Slack
- `Result/Output/project/project_1_시민문의봇/data/answer_templates.xlsx` — LLM 프롬프트에 Few-shot으로 삽입

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    Dify부터 단계적으로 만드세요:

    1. **분류기만** — 민원을 입력받아 카테고리 1개만 출력. 이것부터 확인.
    2. **답변 생성기 추가** — 분류된 카테고리 변수를 프롬프트에 전달해 답변 템플릿 기반 초안 생성.
    3. **구조화 출력** — Dify의 "JSON 출력" 기능으로 `{category, department, draft}` 형태로 반환.

    **각 단계마다 작동을 확인**한 뒤 다음 단계로. 처음부터 완성형을 만들려 하지 마세요.

??? tip "힌트 2: 도구 활용이 막힌다면"
    **Dify 분류 프롬프트 예시**:
    ```
    당신은 민원 분류 전문가입니다. 다음 민원을 아래 7개 카테고리 중 하나로 분류하세요.
    카테고리: 도로·교통 / 환경·위생 / 시설관리 / 행정문의 / 복지·보건 / 세무 / 기타

    민원: {{input.minwon}}
    출력 형식: {"category": "...", "confidence": 0.0~1.0}
    ```

    **n8n에서 Dify 호출**은 Lab 14에서 한 것을 그대로 재사용. Credentials만 준비되면 HTTP Request 노드 하나면 됩니다.

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    **Dify Chatflow 최종 출력 예시**:
    ```json
    {
      "category": "도로·교통",
      "department": "도로과",
      "draft": "안녕하십니까. 소중한 민원을 제기해주셔서 감사합니다. 신고해주신 포트홀 관련하여 담당 부서에서 현장 확인 후 조치하도록 하겠습니다. 처리 기간은 접수일로부터 영업일 기준 3일 이내입니다. 불편을 드려 죄송하며, 빠른 조치로 보답드리겠습니다."
    }
    ```

    **n8n Switch 노드 예시**:
    - Rule 1: `{{ $json.category }} == "도로·교통"` → 도로과 메일
    - Rule 2: `{{ $json.category }} == "환경·위생"` → 환경과 메일
    - ...

## Phase 3. 확장하기 (Extend) — 약 25분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    30건의 샘플 민원을 모두 Chatflow에 넣어보고, **오분류된 건**과 **답변 초안이 어색한 건**을 각각 2개 이상 찾아 프롬프트나 분기 로직을 개선하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    다음 질문으로 점검해보세요:

    - "카테고리가 겹치는 민원(예: 불법주차로 인한 소음)은 어떻게 처리되었는가?"
    - "답변 초안에 민원인 이름이나 구체 정보가 빠져 있지 않은가?"
    - "지나치게 딱딱하거나, 반대로 부적절하게 가벼운 톤은 아닌가?"

    ChatGPT에 초안 10개를 보여주고 "공공기관 답변으로 적절하지 않은 부분이 있다면 지적해주세요"라고 물어보는 것도 효과적입니다.

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    민원뿐 아니라 **시민 제안** 텍스트도 같은 시스템에서 처리하도록 확장하세요. 제안의 경우 `답변 초안` 대신 **"담당부서 후보 3개 + 배정 근거"**를 반환하도록 Chatflow를 분기하세요.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    입력 단계에서 **"민원 / 제안"** 타입을 먼저 구분(Dify의 분기 노드 또는 앞단 LLM)하고, 타입별로 다른 LLM 프롬프트를 태우는 방식이 가장 깔끔합니다.

    제안 테스트 데이터: `Result/Output/project/project_1_시민문의봇/data/sample_proposals.xlsx` (샘플 제안 15건). 제안의 경우 `departments.xlsx`의 "연관키워드"를 Dify 지식베이스로 등록해, 제안 내용과 키워드 유사도로 담당부서 후보를 추천하게 할 수 있습니다.
