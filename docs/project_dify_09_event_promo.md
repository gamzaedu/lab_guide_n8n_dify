# Project 09. 행사 홍보 자동 파이프라인

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 행사 홍보 자동 파이프라인 |
| 소요 시간 | 2교시 (100분) |
| 선행 Lab | **Lab 07. 첫 Workflow 문서 요약기**, **Lab 09. Iteration 다중 항목 처리**, **Lab 10. HTTP Request 노드** |
| 사용 도구 | Dify (Workflow), ChatGPT, 바이브코딩 (Claude/Cursor) |

📌 **진행 흐름**

`Phase 1 정의하기 (약 20분)` → `Phase 2 구현하기 (약 55분)` → `Phase 3 확장하기 (약 25분)`

> 이 프로젝트는 **Lab 07. 첫 Workflow 문서 요약기**, **Lab 09. Iteration 다중 항목 처리**, **Lab 10. HTTP Request 노드**에서 익힌 Workflow 구조 설계, 다중 항목 반복 생성, 외부 연동 기초를 통합 적용합니다.

### 시나리오

> 당신은 지역 행사·축제를 담당하는 홍보 주무관입니다.
> 매달 크고 작은 행사가 열리는데, 같은 행사의 홍보 콘텐츠를 SNS용·카카오톡 채널용·웹 안내페이지용으로 각각 손수 작성하고 있습니다. 톤과 스타일을 맞추는 데도 시간이 많이 듭니다.
> 행사 정보를 한 번만 입력하면 여러 채널용 콘텐츠가 한 번에 생성되고, 바이브코딩으로 만든 간단한 안내 페이지에까지 자동 삽입되는 **end-to-end 파이프라인**을 구축해야 합니다.

### 최종 산출물

- 작동하는 Dify Workflow (행사 입력 → SNS 카드뉴스 문구 + 카톡 메시지 + 랜딩페이지 카피)
- 바이브코딩으로 제작한 HTML 랜딩페이지 (생성된 카피가 주입된 완성본)
- 설계 문서 (채널별 톤·길이 규칙, 재사용 가능한 프롬프트 구조)

## Phase 1. 정의하기 (Define) — 약 20분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 Workflow 문서 요약과 Iteration 처리를 바탕으로, 지금부터는 스스로 문제를 정의합니다.

!!! abstract "과제"
    홍보 대상 **채널 3개 이내**(예: SNS/카카오톡/랜딩페이지)를 선정하고, 각 채널의 **톤·길이·필수 요소**를 규칙으로 정리하세요.

**요구사항 정리**

- [ ] 홍보 채널 3개가 선정되었고 각 채널의 독자가 명확하다
- [ ] 채널별 글자 수 가이드라인이 정해졌다
- [ ] 필수 포함 요소(일시·장소·신청방법·문의처)가 모든 채널에 반영되도록 정의되었다
- [ ] 기관 톤앤매너 규칙(금지어·선호 표현)이 작성되었다

**활용 도구 & 데이터**

- `Result/Output/project/project_3_행사홍보/data/sample_events.xlsx` — 샘플 행사 정보 5건
- `Result/Output/project/project_3_행사홍보/data/banned_preferred_words.xlsx` — 금지어·선호어 15건 (기관 톤앤매너)
- `Result/Output/project/project_3_행사홍보/data/channel_spec.xlsx` — 채널별 규격 (글자수·해시태그·필수요소)
- `Result/Output/project/project_3_행사홍보/data/past_promotions.xlsx` — 과거 홍보 사례 10건 (Few-shot)
- `Result/Output/project/project_3_행사홍보/data/landing_template.html` — 바이브코딩 출발점 HTML

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    `past_promotions.xlsx`의 10건 사례를 읽어보세요. 같은 성격의 행사라도 SNS·카카오톡·랜딩페이지에서 **다른 길이와 톤**을 쓰는지 확인할 수 있습니다. 이 차이를 규칙화한 것이 Phase 1의 산출물입니다.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에 다음과 같이 물어보세요:

    "나는 공공기관 홍보 담당자이고, 행사 홍보 콘텐츠를 채널별로 자동 생성하려 한다. SNS 카드뉴스·카카오톡 채널 메시지·랜딩페이지 본문, 이 세 채널의 독자 특성과 적절한 글자 수, 톤 차이를 정리해줘."

    `channel_spec.xlsx`의 6개 채널 규격(글자수·해시태그·필수요소)을 그대로 워크플로우 분기의 조건으로 쓰면 됩니다. 톤은 `banned_preferred_words.xlsx`의 선호어/금지어를 프롬프트에 주입하세요.

## Phase 2. 구현하기 (Build) — 약 55분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    행사 정보를 입력받아 **3개 채널용 콘텐츠를 한 번에 생성**하는 Dify Workflow를 만들고, 바이브코딩(Claude/Cursor)으로 HTML 랜딩페이지 템플릿을 제작해 Workflow 출력을 주입하세요.

**마일스톤**

☐ Dify Workflow 시작 노드에 행사 입력 변수(행사명·일시·장소·대상·주요내용 등)가 정의되었다
☐ SNS용·카카오톡용·랜딩페이지용 LLM 노드 3개가 병렬로 실행된다
☐ 각 LLM 노드가 `banned_preferred_words.xlsx`와 `past_promotions.xlsx`를 Few-shot/제약조건으로 참조한다
☐ Workflow 최종 출력이 `{sns_text, kakao_text, landing_copy}` 구조로 반환된다
☐ 바이브코딩으로 `landing_template.html`을 완성하고, 생성된 `landing_copy`를 삽입해 실제 페이지가 렌더링된다

**활용 도구 & 데이터**

- Dify Workflow: 시작 → [LLM(SNS) / LLM(카톡) / LLM(랜딩)] 병렬 → 종료
- 바이브코딩: Claude Desktop 또는 Cursor로 `landing_template.html` 수정·완성
- `sample_events.xlsx`의 행사 1건을 실제 입력으로 사용

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    **한 채널부터** 완성하세요. SNS 카드뉴스 노드 하나만 먼저 완벽히 작동시킨 후, 나머지 2개를 복제해 프롬프트만 수정하는 방식이 가장 빠릅니다.

    바이브코딩도 마찬가지. `landing_template.html`을 그대로 브라우저로 열어보고, 그 다음 Claude/Cursor에 "이 HTML의 {{행사명}}, {{일시}} 변수에 다음 값을 넣어줘"라고 지시하세요.

??? tip "힌트 2: 도구 활용이 막힌다면"
    **Dify LLM 프롬프트 템플릿** (SNS 카드뉴스용):
    ```
    당신은 공공기관 홍보 담당자입니다. 아래 선호어/금지어와 예시를 참고해,
    행사 정보를 바탕으로 SNS 카드뉴스 3컷 문구를 작성하세요.

    선호어/금지어 (banned_preferred_words.xlsx):
    {{brand_words}}

    예시 (past_promotions.xlsx의 Few-shot):
    {{example_past_cases}}

    행사 정보:
    행사명: {{event_name}}
    일시: {{event_date}}
    ...

    출력 형식 (JSON):
    {"card1": "...", "card2": "...", "card3": "..."}
    ```

    **바이브코딩에서 Claude에게 이렇게 부탁하세요**:
    "이 HTML 템플릿의 `{{행사명}}` 같은 플레이스홀더를 실제 값으로 치환한 완성 페이지를 만들어줘. 입력 JSON은 아래와 같아: ..."

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    **Dify 최종 출력 JSON 예시**:
    ```json
    {
      "sns_cards": ["청년 창업가, 우리 시가 함께합니다.", "하루 동안 이 모든 것을...", "참가비 무료 · 선착순 200명"],
      "kakao_message": "🌱 청년 창업가 여러분께 ○○시가 특별한 하루를 준비했습니다...",
      "landing_copy": {
        "hero_title": "청년 창업가, 우리 시가 함께합니다.",
        "hero_subtitle": "○○시 청년 창업 페어 2026",
        "body": "..."
      }
    }
    ```

    이 JSON을 `landing_template.html`의 플레이스홀더에 그대로 주입하면 완성 페이지가 됩니다.

## Phase 3. 확장하기 (Extend) — 약 25분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    `sample_events.xlsx`의 서로 다른 행사 3건으로 Workflow를 돌려보고, **톤의 일관성**과 **필수 정보 누락 여부**를 점검해 프롬프트를 개선하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    다음 질문으로 점검해보세요:

    - "행사마다 톤이 들쭉날쭉하지 않은가? (어르신 대상인데 너무 힙한 표현 등)"
    - "일시·장소·신청방법이 모든 채널에 빠짐없이 들어갔는가?"
    - "금지어가 섞여 들어가지 않았는가?"

    동료 1명에게 결과물을 보여주고 "공식 홍보물로 바로 쓸 수 있겠느냐"라고 물어보는 것이 가장 빠른 QA입니다.

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    **n8n을 앞단에 연결**해, Google Sheets에 행사 정보가 새로 입력되면 자동으로 Dify Workflow를 호출하고 결과를 Sheets에 다시 기록하도록 파이프라인을 확장하세요.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    파이프라인은 다음과 같습니다:

    ```
    [Google Sheets Trigger] → [HTTP Request(Dify API 호출)] → [Sheets에 결과 기록]
    ```

    Lab 14(n8n에서 Dify API 호출)를 그대로 재사용. 추가로 바이브코딩으로 만든 HTML 템플릿에 결과를 주입해 정적 페이지를 배포(GitHub Pages)하는 것까지 자동화하면 진짜 end-to-end 완성입니다.
