# Project 10. 뉴스 여론 모니터링 대시보드

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 뉴스 여론 모니터링 대시보드 |
| 소요 시간 | 3교시 (150분) |
| 선행 Lab | **Lab 02. Trigger 노드 비교**, **Lab 04. HTTP Request 노드**, **Lab 14. Pagination**, **Lab 18. Google Sheets**, **Lab 21. OpenAI 노드** |
| 사용 도구 | n8n, HTTP API(뉴스 RSS 또는 오픈 검색 API), Google Sheets, ChatGPT(OpenAI API) |

📌 **진행 흐름**

`Phase 1 정의하기 (약 30분)` → `Phase 2 구현하기 (약 80분)` → `Phase 3 확장하기 (약 40분)`

> 이 프로젝트는 **Lab 02**, **Lab 04**, **Lab 14**, **Lab 18**, **Lab 21**에서 익힌 Schedule, HTTP Request, Pagination, Sheets 쓰기, GPT 분류를 통합 적용합니다.

### 시나리오

> 당신은 홍보·정책 담당 부서에서 "우리 기관·사업에 대한 여론"을 관찰해야 하는 담당자입니다.
> 매일 손으로 네이버·다음 뉴스를 검색하고 엑셀에 옮겨 분석하는 일이 반복되고 있어, 놓치는 기사도 많고 트렌드를 보지 못합니다.
> 지정 키워드로 최신 기사(또는 공개 SNS 게시물)를 자동 수집해, GPT로 감성(긍정/중립/부정)과 주제(정책 공감/불만/사실 전달/기타)를 분류하고, 날짜별 집계와 원문 목록을 Sheets 대시보드에 자동 축적하는 워크플로우를 만드세요.

### 최종 산출물

- 작동하는 n8n 워크플로우 (Schedule → HTTP Pagination → 중복 제거 → GPT 분류 → Sheets 적재)
- Google Sheets 대시보드 (raw 탭 + daily 탭 최소 3일치 데이터)
- 키워드·분류 체계·프롬프트 정의 메모

## Phase 1. 정의하기 (Define) — 약 30분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 HTTP Request와 Pagination을 바탕으로, 지금부터는 "무엇을 관찰할지와 어떻게 분류할지"를 스스로 정의합니다.

!!! abstract "과제"
    관찰할 키워드 2~3개(예: 기관명·주요 정책 명칭)와 감성·주제 분류 체계를 정의하고, 대시보드가 한눈에 보여줘야 할 지표 3~4개를 정하세요.

**요구사항 정리**

- [ ] 관찰 키워드 2~3개가 선정되었다 (너무 흔하지 않고, 너무 좁지도 않은 수준)
- [ ] 감성 체계가 정의되었다 (예: positive / neutral / negative)
- [ ] 주제 카테고리 3~4개가 정의되었다
- [ ] 중복 제거 기준이 정해졌다 (URL, 또는 제목 해시)
- [ ] 대시보드 지표 3~4개가 정해졌다 (일자별 건수, 감성 분포, 주제 분포, 인기 출처 등)

**활용 도구 & 데이터**

- 뉴스 검색용 공개 API 또는 RSS 2개 (NAVER Open API, Google News RSS 등 접근 가능한 것)
- Google Sheets: `raw` 탭(수집 원본) + `daily` 탭(일자별 집계)
- ChatGPT: 분류 체계 점검, Few-shot 예시 작성

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    지난 2주간 우리 기관이 언급된 기사 중 인상 깊었던 3건을 꺼내보세요.
    각 건을 어느 감성·주제로 라벨링하고 싶은지가 분류 체계의 시작점입니다.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에게 다음과 같이 물어보세요:
    "공공기관 홍보 담당자를 위한 뉴스 감성·주제 분류 체계를 제안해줘. 감성 3단계, 주제 4~5개, 각 카테고리의 판단 기준과 Few-shot 예시 1개씩."

## Phase 2. 구현하기 (Build) — 약 80분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    Schedule로 매 6시간(또는 일 1회) 실행되는 워크플로우를 만드세요. 키워드로 API를 호출해 최근 기사를 수집하고, 중복을 제거한 뒤 GPT로 감성·주제를 분류해 Sheets `raw` 탭에 적재하고, 같은 날짜의 집계를 `daily` 탭에 갱신합니다.

**마일스톤**

☐ Schedule Trigger가 정해진 주기로 실행된다
☐ HTTP Request 노드로 검색 결과 JSON이 받아지고 Pagination으로 다음 페이지가 연결된다
☐ Code 또는 Sheets 조회로 이미 수집된 URL이 중복 제거된다
☐ OpenAI 노드가 각 기사를 {sentiment, topic, reason} JSON으로 분류한다
☐ Sheets `raw` 탭에 수집 날짜·제목·출처·URL·감성·주제 컬럼이 한 행씩 적재된다
☐ Sheets `daily` 탭에 오늘 날짜 행이 생성/갱신되고 감성·주제 분포가 기록된다
☐ 최소 3일 누적 후 대시보드에서 간단한 추세 확인이 가능하다

**활용 도구 & 데이터**

- n8n: Schedule / HTTP Request / Code / OpenAI / Google Sheets
- Sheets: `raw` / `daily` 두 탭 분리
- ChatGPT: 프롬프트 + Few-shot 설계

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    Pagination·중복제거·GPT 분류를 한꺼번에 꺼내지 마세요. 아래 순서로 쌓습니다.

    1. HTTP Request 1회 호출 → 기사 10건 받기
    2. Sheets `raw`에 바로 적재 (분류·중복 생략)
    3. GPT 분류를 추가, 결과 컬럼 덧붙이기
    4. 중복 제거 로직 추가
    5. 마지막에 Pagination과 `daily` 집계 추가

??? tip "힌트 2: 도구 활용이 막힌다면"
    중복 제거 Code 예시:
    ```js
    const existing = new Set(($items('Read raw') || []).map(i => i.json.url));
    return items.filter(i => !existing.has(i.json.url));
    ```

    daily 집계 Expression 예시(간단히):
    ```
    pos = {{ items.filter(i => i.json.sentiment === 'positive').length }}
    neg = {{ items.filter(i => i.json.sentiment === 'negative').length }}
    ```

    라이트한 감성 분류는 OpenAI 응답 형식을 JSON으로 고정하고 temperature를 낮게(예: 0.2) 두면 안정적입니다.

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    GPT 분류 프롬프트 골격 (Few-shot 포함):
    ```
    너는 한국어 뉴스 분류기다. 반드시 JSON만 출력한다.
    스키마: {"sentiment":"positive|neutral|negative","topic":"정책 공감|불만|사실 전달|기타","reason":"한 줄 근거"}

    예시 1
    기사: "...우수 사례로 선정..."
    출력: {"sentiment":"positive","topic":"정책 공감","reason":"수상·선정 보도"}

    예시 2
    기사: "...주민 반발 거세..."
    출력: {"sentiment":"negative","topic":"불만","reason":"주민 반대 직접 인용"}

    이제 아래를 분류하라.
    기사 제목: {{title}}
    본문 요약: {{snippet}}
    ```

    `daily` 탭 컬럼 예:
    | 날짜 | 총건수 | positive | neutral | negative | 최다 주제 |

## Phase 3. 확장하기 (Extend) — 약 40분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    수집된 30건을 직접 손으로 다시 분류해보고, GPT와 다른 결과가 나온 건 2건의 원인을 분석해 프롬프트 또는 Few-shot 예시를 개선하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    일치하지 않는 이유는 보통 다음 중 하나입니다.

    - 감성·주제 정의가 겹쳐 있다 (예: "불만"과 "negative"가 항상 같이 가야 하는지)
    - Few-shot 예시가 짧아 모호한 기사에 약하다 → 경계 사례 예시 1개 추가
    - 단순 제목만 보고 판단하고 있다 → 본문 스니펫도 함께 주입
    - 모델 출력이 스키마를 가끔 어긴다 → Response Format을 JSON으로 강제

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    다음 중 하나를 추가하세요. (1) 부정 급증 감지 — 당일 negative 비율이 평균 대비 +30%p 오르면 담당자 Slack/Gmail 경보, (2) 주간 요약 리포트 — GPT로 "이번 주 키 토픽 3가지" 자동 생성, (3) 키워드 2개 비교 대시보드 — 서로 다른 사업명·정책명의 여론 흐름을 한 탭에서 비교.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    "수집 → 분류 → 적재"의 뼈대는 고객 리뷰·SNS 모니터링·경쟁기관 동향 등에 그대로 재활용됩니다.
    키워드와 분류 체계만 바꾸면 같은 파이프라인이 다른 도메인의 여론 대시보드가 됩니다.
