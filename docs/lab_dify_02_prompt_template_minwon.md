??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"

    **[의미]**

    수업에서 배운 내용을 직접 적용해보는 단계입니다.

    **[지침]**

    - 가능한 직접 손으로 타이핑해보고 직접 생각해봅니다.
        - 5분 정도는 AI 에서 손을 떼고 펜을 들어보세요!
    - 한번의 완벽한 결과보다 다양한 시도를 통한 점진적 개선을 추구합니다.

!!! abstract "과제"
    아래 **4블록 프롬프트 템플릿**(역할 / 답변 규칙 / 출력 포맷 / 예시)을 그대로 System Prompt로 입력해 민원 1차 응대 챗봇을 만들고, 시민 민원 3건을 테스트하세요.

**활용 도구 & 데이터**

- **도구:** Dify Chatbot (앱 이름: `Lab 02 - 민원 답변 도우미`)
- **4블록 System Prompt 템플릿:**

```
## 역할
당신은 "○○시청 민원실" 1차 응대 상담사입니다.
시민이 민원을 접수하기 전, 어떤 부서·어떤 서류가 필요한지 안내합니다.

## 답변 규칙
1) 민원 내용을 듣고 **담당 부서**, **필요 서류**, **예상 처리 기간**을 알려줍니다.
2) 답변은 항상 아래 "출력 포맷"을 따릅니다.
3) 확실치 않으면 "확인이 필요합니다. 1층 종합 안내실로 문의 바랍니다" 로 답합니다.
4) 친절·정중한 말투를 유지합니다.

## 출력 포맷
- 담당 부서: ○○과 (○층)
- 필요 서류: 1) ... 2) ... 3) ...
- 처리 기간: ○일 이내
- 한줄 안내: ...

## 예시
Q: 자동차 등록증을 잃어버렸어요.
A:
- 담당 부서: 차량등록과 (3층)
- 필요 서류: 1) 신분증 2) 도장 3) 수수료 3,000원
- 처리 기간: 당일 발급
- 한줄 안내: 분실 시 즉시 재발급을 권장드립니다.
```

- **테스트 민원 3건:**
    1. "전입신고하러 왔는데 뭐가 필요해요?"
    2. "임산부 혜택 신청하려고요."
    3. "개 키우면 등록해야 한다던데 어디서 하나요?"

**실행 단계:**

1. Chatbot 신규 앱 생성 → 이름 `Lab 02 - 민원 답변 도우미`
2. Instructions에 위 4블록 템플릿 전체 복사·붙여넣기
3. Opening Statement: `안녕하세요. 민원 1차 안내 도우미입니다. 어떤 민원이 있으신가요?`
4. Temperature: `0.2` (포맷 일관성 우선)
5. Debug & Preview에서 3건 테스트

??? note "예시"

    > **Q: 전입신고하러 왔는데 뭐가 필요해요?**
    >
    > **A:**
    >
    > - 담당 부서: 주민등록과 (2층)
    > - 필요 서류: 1) 신분증 2) 임대차계약서 또는 등기부등본 3) 세대주 동의서(별도 세대 합가 시)
    > - 처리 기간: 당일 처리
    > - 한줄 안내: 전입일로부터 14일 이내 신고하지 않으면 과태료가 부과됩니다.
    >
    > **Q: 임산부 혜택 신청하려고요.**
    >
    > **A:**
    >
    > - 담당 부서: 가족복지과 (4층)
    > - 필요 서류: 1) 신분증 2) 임신확인서(산부인과 발급) 3) 통장 사본
    > - 처리 기간: 신청 즉시 바우처 발급
    > - 한줄 안내: 국민행복카드가 없으면 함께 신청 가능합니다.
    >
    > **확인 포인트:** 세 답변 모두 **네 줄 포맷(부서/서류/기간/안내)** 이 동일하게 유지되는지 확인합니다. 포맷이 흔들리면 System Prompt의 "출력 포맷" 블록이 약하다는 신호입니다.

### Step 2. 분석과 성찰 (Analyzing & Reflecting)

??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"
        객관적인 피드백을 구하되, 판단은 스스로 합니다.

    **[의미]**

    Step 1 에서 만든 결과물을 한 발 물러서서 객관적으로 바라보는 단계입니다.
    혼자서는 보이지 않던 것들이, 다른 시선을 통해 드러납니다.

    **[지침]**

    - 1단계 작업물에 대해, 객관적인 피드백을 얻습니다
    - 다른 탐색하기 결과물을 확인합니다
        - 다른 동료의 작업 결과물 | AI 의 생성 결과물 | 웹검색 등
    - AI 를 사용할 땐 반드시 비판적인 자세를 유지합니다.
        - AI 가 생성한 결과물 및 지침은 그대로 받아들이지 않고, 의심합니다.

!!! abstract "과제"
    **예시(Few-shot) 블록을 제거한 버전(A)** 과 **포함한 버전(B)** 으로 같은 3건을 실행해, 포맷 일관성·정확도를 비교 분석하세요. 어느 요소가 답변 품질에 가장 큰 영향을 주는지 표로 정리하세요.

**활용 도구 & 데이터**

- **도구:** Step 1 챗봇 (Instructions 만 두 버전으로 교체)
- **비교 설계:**

| 버전 | System Prompt 구성 |
|------|--------------------|
| A (예시 제거) | 역할 + 답변 규칙 + 출력 포맷 (예시 블록 삭제) |
| B (예시 포함) | 위 4블록 전체 (Step 1 원본) |

- **동일 질문 3건:** Step 1과 동일

**측정 항목:**

| 항목 | 측정 방법 |
|------|----------|
| 포맷 일관성 | 답변 3개가 모두 네 줄 포맷을 따르면 ✅ |
| 항목 누락 | 4개 항목(부서/서류/기간/안내) 중 빠진 것 수 |
| 정확도 | 부서명·서류가 실제 업무와 일치하는지 ○/△/× |

**비교 방법:**

1. Instructions을 A 버전으로 교체 → 3건 실행 → 답변 캡처
2. B 버전으로 교체 → 같은 3건 실행 → 답변 캡처
3. 아래 비교표를 작성

**예시**

> **비교 분석표 — A(예시 없음) vs B(예시 포함)**
>
> | 질문 | A 포맷 일관성 | A 누락 | B 포맷 일관성 | B 누락 |
> |------|--------------|--------|--------------|--------|
> | 전입신고 | △ (기간 줄글로 섞임) | 1개 | ✅ | 0개 |
> | 임산부 혜택 | ✅ | 0개 | ✅ | 0개 |
> | 동물등록 | × (줄글 답변) | 3개 | ✅ | 0개 |
>
> **분석 결과:**
>
> - 예시(Few-shot)가 없어도 **쉬운 질문(임산부 혜택)** 은 포맷을 따르지만, **생소한 주제(동물등록)** 에서는 줄글로 흩어짐
> - **Few-shot 1개만 추가해도** 포맷 일관성이 크게 올라감 — 가성비가 가장 높은 개선 방법
> - 답변 규칙만 텍스트로 설명하면 모델이 "해석"하지만, 예시를 주면 "모방"하므로 훨씬 안정적
> - 실무 적용 팁: 같은 부서의 **대표 민원 3건을 예시로 박제**해두면 정확도·일관성 모두 향상

### Step 3. 창조하기 (Creating)

??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"
        완성품이 아니라, 작동하는 시작품을 만듭니다.

    **[의미]**

    앞서 얻은 통찰을 바탕으로, 나만의 결과물을 만들어 봅니다.

    **[지침]**

    - 2단계에서 얻은 피드백을 원하는 만큼 반영합니다.
        - 중요한 점은, 받아들일 수 있는 만큼만 받아들이는 것입니다.
    - 내 상황과 스타일에 맞게 최적화된 결과물을 만들어 봅니다.
    - 처음부터 완성하는 것이 아닌 계속해서 개선해 나가는 것입니다.
        - 가장 작은 단위부터 프로토타입으로 만들어 보세요

!!! abstract "과제"
    **본인 부서의 민원 1차 응대 챗봇**을 완성하세요. 4블록 구조(역할/규칙/포맷/예시)를 모두 포함하고, Few-shot 예시 **2개 이상**, Opening Statement, Suggested Questions 3개까지 배치합니다. 테스트 민원 5건에서 포맷 일관성 100%를 달성하면 Publish 합니다.

**활용 도구 & 데이터**

- **도구:** Dify Chatbot
- **완성 체크리스트:**

| 항목 | 필수 |
|------|------|
| System Prompt 4블록 구조 | ✅ |
| Few-shot 예시 2개 이상 | ✅ |
| Opening Statement | ✅ |
| Suggested Questions 3개 | ✅ |
| 테스트 민원 5건 포맷 일관성 | ✅ |
| Publish 후 공개 URL 확보 | ✅ |

**완성 순서:**

1. 본인 부서에서 자주 오는 민원 유형을 5개 정리(대표 민원 기반)
2. 그중 2개를 Few-shot 예시로 System Prompt에 박제
3. Opening Statement와 Suggested Questions 작성
4. 나머지 3개 민원으로 테스트 → 포맷 일관성 확인
5. Publish → 공개 URL 복사 → 동료 1인에게 보내 피드백 받기

**예시**

> **완성 챗봇: "환경과 민원 1차 안내"**
>
> **System Prompt (발췌):**
>
> ```
> ## 역할
> 당신은 ○○시청 "환경과" 민원 1차 안내 상담사입니다.
> (하략)
>
> ## 예시
> Q: 재활용 분리배출 어떻게 하나요?
> A:
> - 담당 부서: 환경과 자원순환팀 (5층)
> - 필요 서류: 없음 (현장 상담)
> - 처리 기간: 즉시 안내
> - 한줄 안내: 매주 화·목 오전에 분리배출 교육을 운영합니다.
>
> Q: 대형폐기물 스티커는 어디서 사요?
> A:
> - 담당 부서: 환경과 청소관리팀 (5층)
> - 필요 서류: 없음 (온라인/동주민센터 구매)
> - 처리 기간: 스티커 구매 후 3일 이내 배출
> - 한줄 안내: 시청 홈페이지 "대형폐기물" 메뉴에서도 신청 가능합니다.
> ```
>
> **Opening Statement:** `안녕하세요, 환경과 민원 안내 도우미입니다. 어떤 환경 민원이 있으신가요?`
>
> **Suggested Questions:**
>
> - 분리배출 요일이 어떻게 되나요?
> - 대형폐기물 스티커 구매 방법이 궁금해요.
> - 층간소음 민원은 어디로 접수하나요?
>
> **테스트 결과 5/5 통과, Publish URL 확보.**

**활용 도구 & 데이터**

자유롭게 활용

**예시**

> - **Few-shot 예시를 1개, 2개, 3개로 늘리면 품질은 어떻게 변화할까?**
>     - 같은 질문 10건에 대해 예시 개수만 바꿔 실행하고, 포맷 일관성·정확도·답변 속도 변화를 추적해보세요. "가성비 좋은 예시 개수"가 몇 개인지 근거를 찾아보세요.
> - **예시를 "잘못된 답변"과 "올바른 답변"의 쌍으로 주면 어떨까?**
>     - `X: 잘못된 예` / `O: 올바른 예` 구조를 실험해보세요. 모델이 피해야 할 패턴을 더 빨리 학습하는지 관찰해보세요.
> - **출력 포맷을 Markdown 표 형태로 강제하면 답변 품질이 더 좋아질까?**
>     - 현재 bullet 포맷을 `| 항목 | 내용 |` 표 포맷으로 바꿔 실험해보세요. 가독성과 정확도 변화가 있는지 살펴보세요.
> - **민원 질문이 매우 짧거나(1~2단어) 너무 모호할 때** 챗봇은 어떻게 반응할까?
>     - "등본", "과태료" 같은 1단어 질문 20개를 던져 fallback 규칙이 잘 작동하는지 측정하고, 어떤 프롬프트 수정이 필요한지 정리해보세요.
