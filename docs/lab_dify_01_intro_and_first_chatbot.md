# Lab 1. Dify 시작하기 & 첫 챗봇 만들기

## 개요

| 항목 | 내용 |
|------|------|
| 학습 목표 | Dify Cloud에서 Chatbot 앱을 만들고, System Prompt를 작성해 원하는 역할의 첫 챗봇을 동작시킬 수 있다 |
| 핵심 기능 | Studio, 앱 유형(Chatbot), System Prompt, Model Provider |
| 소요 시간 | 약 40분 |
| 사전 준비 | Dify Cloud 계정, 사용 가능한 LLM 모델(OpenAI 등) 연결 완료 |
| 난이도 | ★☆☆☆☆ |

!!! info "참고"
    Dify Cloud 화면은 버전에 따라 메뉴 위치와 이름이 조금씩 달라질 수 있습니다. 본 가이드는 2026년 기준으로 작성되었으며, 실제 화면과 다르면 유사한 명칭의 메뉴를 찾아 진행하세요.

### Step 1. 탐색하기 (Exploring)

??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"
        남이 알려준 정답보다 나의 시도가 더 가치 있습니다.

    **[의미]**

    수업에서 배운 내용을 직접 적용해보는 단계입니다.

    **[지침]**

    - 가능한 직접 손으로 타이핑해보고 직접 생각해봅니다.
        - 5분 정도는 AI 에서 손을 떼고 펜을 들어보세요!
    - 한번의 완벽한 결과보다 다양한 시도를 통한 점진적 개선을 추구합니다.
    - 실패를 두려워하지 마세요. 빠른 실패는 오히려 빠른 성장을 가져옵니다!

!!! abstract "과제"
    Dify Studio에서 **Chatbot 앱**을 하나 만들고, System Prompt로 "우리 시청의 부서 안내 도우미" 역할을 부여하세요. 미리보기(디버그) 패널에서 최소 3가지 질문을 던져 답변을 받아보세요.

**활용 도구 & 데이터**

- **도구:** Dify Cloud (Studio)
- **앱 이름:** `Lab 01 - 부서 안내 도우미`
- **사용 모델:** OpenAI `gpt-4o-mini` 또는 `gpt-3.5-turbo` (비용 저렴한 모델 권장)

**앱 생성 단계:**

1. **Studio 진입**
    - 상단 메뉴 **[Studio]** → 우측 상단 **[Create App]** → **[Create from Blank]** 선택
2. **앱 유형 선택**
    - 4가지 중 **Chatbot** 선택
    - App Name: `Lab 01 - 부서 안내 도우미`
    - 설명(선택): `시청 방문자에게 부서 위치와 담당업무를 안내하는 챗봇`
3. **System Prompt 작성**
    - 좌측 패널 **Instructions** 영역에 아래 프롬프트 입력

```
당신은 "○○시청 부서 안내 도우미"입니다.
시청 방문자가 민원·업무와 관련해 어느 부서로 가야 하는지 물어보면,
다음 원칙으로 답변하세요.

1) 친절하고 정중한 말투를 사용합니다.
2) 가능한 부서명과 층수, 연락처를 함께 안내합니다.
3) 정확히 모르는 경우에는 "종합 안내(1층 민원실)로 문의 바랍니다"라고 답합니다.
4) 답변은 3문장 이내로 간결하게 합니다.
```

4. **모델 및 파라미터 설정**
    - 우측 상단 모델 선택 → `gpt-4o-mini`
    - Temperature: `0.3` (정확도 우선)
5. **대화 실행**
    - 우측 Debug & Preview 영역에 아래 질문을 차례로 입력해 답변 확인
        - "출생신고는 어디서 하나요?"
        - "건축 허가 관련 상담은요?"
        - "주차 과태료는 어디로 문의해요?"

??? note "예시"

    > **System Prompt 입력 결과 (Dify 미리보기 답변)**
    >
    > | 질문 | 챗봇 답변 |
    > |------|----------|
    > | 출생신고는 어디서 하나요? | "출생신고는 1층 **가족관계등록과** 에서 접수 가능합니다. 전화는 02-000-0001 입니다. 신분증과 출생증명서를 지참해주세요." |
    > | 건축 허가 관련 상담은요? | "건축 허가·신고 관련 업무는 3층 **건축과** 입니다. 사전 상담은 02-000-0003 으로 예약 가능합니다." |
    > | 주차 과태료는 어디로 문의해요? | "주차 과태료는 4층 **교통행정과** 담당입니다. 02-000-0004 로 문의해주세요." |
    >
    > **확인 포인트:** 세 답변 모두 (1) 부서명, (2) 층수, (3) 연락처가 포함되고 3문장 이내로 간결한지 확인합니다. 형식이 일정하면 System Prompt가 제대로 작동한 것입니다.

### Step 2. 분석과 성찰 (Analyzing & Reflecting)

??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"
        객관적인 피드백을 구하되, 판단은 스스로 합니다.

    **[의미]**

    Step 1 에서 만든 결과물을 한 발 물러서서 객관적으로 바라보는 단계입니다.
    혼자서는 보이지 않던 것들이, 다른 시선을 통해 드러납니다.

    **[지침]**

    - 1단계 작업물에 대해, 객관적인 피드백을 얻습니다
    - 다른 탐색하기 결과물을 확인합니다
        - 다른 동료의 작업 결과물 | AI 의 생성 결과물 | 웹검색 등
    - AI 를 사용할 땐 반드시 비판적인 자세를 유지합니다.
        - AI 가 생성한 결과물 및 지침은 그대로 받아들이지 않고, 의심합니다.

!!! abstract "과제"
    Step 1의 System Prompt를 **짧은 프롬프트(A)** 와 **상세 프롬프트(B)** 두 버전으로 나눠 같은 질문 3개를 실행하고, 답변 품질·형식·길이를 표로 비교 정리하세요.

**활용 도구 & 데이터**

- **도구:** Step 1에서 만든 챗봇 (Instructions 만 바꿔가며 실행)
- **비교 프롬프트:**

| 버전 | System Prompt |
|------|---------------|
| A (짧음) | `당신은 시청 안내 도우미입니다. 친절하게 답변하세요.` |
| B (상세) | Step 1의 4개 원칙이 모두 포함된 프롬프트 |

- **공통 테스트 질문 3개:**
    1. 출생신고는 어디서 하나요?
    2. 주차 과태료 문의처는?
    3. 도서관 운영시간이 궁금해요.

**변경 방법:**

1. 좌측 Instructions 영역 내용을 A 버전으로 교체 → 상단 **[Publish]** 또는 Debug 창에서 바로 테스트
2. 같은 3개 질문 실행 → 답변 복사
3. Instructions 을 B 버전으로 되돌린 뒤 같은 3개 질문 다시 실행
4. 두 결과를 비교표로 정리

**예시**

> **비교 분석표 — A(짧음) vs B(상세)**
>
> | 질문 | A 답변 | B 답변 | 차이 |
> |------|--------|--------|------|
> | 출생신고 | "가족관계등록과에서 가능합니다." (1문장) | "1층 **가족관계등록과** (02-000-0001) 입니다. 신분증·출생증명서 지참 바랍니다." (2문장) | B는 층수·연락처·준비물 포함 |
> | 주차 과태료 | "교통행정과에 문의하세요." | "4층 **교통행정과** 담당 (02-000-0004) 입니다." | B는 층수·번호 명시 |
> | 도서관 운영시간 | "도서관에 직접 문의해보시기 바랍니다." | "정확한 시간은 확인이 어려워 **1층 종합 안내실** 로 문의 바랍니다." | A는 모름, B는 원칙 4 적용 |
>
> **분석 결과:**
>
> - System Prompt가 **구체적일수록 답변 포맷이 일정**해진다. 공공서비스에서는 형식 일관성이 응답 품질보다 중요할 수 있다.
> - "정확히 모르면 어디로 안내한다"는 **fallback 원칙을 명시해야** 환각(hallucination)을 줄일 수 있다.
> - 짧은 프롬프트는 작성 속도는 빠르지만, **같은 질문에도 답이 매번 달라져** 운영 품질이 들쭉날쭉하다.

### Step 3. 창조하기 (Creating)

??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"
        완성품이 아니라, 작동하는 시작품을 만듭니다.

    **[의미]**

    앞서 얻은 통찰을 바탕으로, 나만의 결과물을 만들어 봅니다.

    **[지침]**

    - 2단계에서 얻은 피드백을 원하는 만큼 반영합니다.
        - 중요한 점은, 받아들일 수 있는 만큼만 받아들이는 것입니다.
    - 내 상황과 스타일에 맞게 최적화된 결과물을 만들어 봅니다.
    - 처음부터 완성하는 것이 아닌 계속해서 개선해 나가는 것입니다.
        - 가장 작은 단위부터 프로토타입으로 만들어 보세요

!!! abstract "과제"
    Step 1~2의 경험을 바탕으로, **본인 부서의 "업무 안내 챗봇"** 을 만드세요. System Prompt에 부서명·담당업무·fallback 원칙을 모두 포함하고, 시민이 자주 묻는 질문 5개에 일관된 답변을 내놓는지 검증하세요. 완성되면 `Publish`를 눌러 공개 URL을 확보합니다.

**활용 도구 & 데이터**

- **도구:** Dify Chatbot 앱 (새로 생성해도 좋고 Step 1 앱을 복제해 써도 좋음)
- **설계 포인트:**

| 항목 | 설명 |
|------|------|
| 앱 이름 | `Lab 01 - <본인 부서명> 업무 안내 챗봇` |
| 부서 | 본인 부서 또는 가상 부서 |
| fallback | "정확히 모르면 ○○로 문의" 문구 필수 |
| 테스트 질문 | 민원인이 실제로 물어볼 법한 질문 5개 |

**완성 순서:**

1. 앱 생성 후 System Prompt에 부서명·주요 업무 3~5개·연락처·fallback을 한 번에 포함
2. Opening Statement(첫 인사말) 작성 — 예: `안녕하세요, ○○과 안내 도우미입니다. 무엇을 도와드릴까요?`
3. Suggested Questions 3개 추가 — 자주 묻는 질문
4. 테스트 질문 5개 실행 후 답변 포맷이 일관적인지 검증
5. 만족스러우면 우측 상단 **[Publish]** → 공개 URL 복사 → 동료에게 공유해 피드백 받기

**예시**

> **완성 챗봇: "민원여권과 업무 안내 도우미"**
>
> **System Prompt:**
>
> ```
> 당신은 ○○시청 "민원여권과" 업무 안내 도우미입니다.
>
> 담당 업무:
> - 주민등록등본·초본 발급
> - 여권 신규발급·재발급
> - 인감증명서 발급
>
> 답변 원칙:
> 1) 친절·정중한 말투를 사용합니다.
> 2) 가능한 "필요 서류 + 처리 시간 + 수수료" 를 함께 안내합니다.
> 3) 담당 업무 밖이면 "죄송합니다, 해당 업무는 1층 종합 안내실로 문의 바랍니다"라고 답합니다.
> 4) 답변은 4문장 이내로 간결하게.
>
> 부서 위치: 2층 202호 / 연락처: 02-000-0010
> ```
>
> **Opening Statement:** `안녕하세요, 민원여권과 안내 도우미입니다. 무엇을 도와드릴까요? 😊`
>
> **Suggested Questions:**
>
> - 여권 재발급에 필요한 서류가 무엇인가요?
> - 인감증명서는 본인만 발급 가능한가요?
> - 등본 발급 수수료는 얼마인가요?
>
> **테스트 결과 (5개 질문 모두 통과):**
>
> | 질문 | 답변 요약 | 통과 여부 |
> |------|----------|----------|
> | 여권 재발급 서류 | 신분증·여권용 사진·기존 여권 + 수수료 5만원 + 1주일 | ✅ |
> | 인감증명서 본인만? | 본인 외 대리발급 가능 (위임장·인감도장 필요) | ✅ |
> | 등본 수수료 | 1통 400원, 무인발급기 무료 | ✅ |
> | 건축 허가? | "1층 종합 안내실 문의" fallback 적용 | ✅ |
> | 처리 소요시간 | 업무별 1~7일 안내 | ✅ |

!!! tip ""
    결과가 만족스럽지 않다면 Step 1이나 Step 2로 돌아가 반복할 수 있습니다.

    **[의미]**

    **[지침]**

    - 왜 좋고 왜 나쁠까? 라는 화두를 갖고, 더 깊은 레벨까지 탐구해 보세요

!!! abstract "과제"
    아래 탐구 질문 중 관심 있는 주제를 골라 직접 실험해보세요. 또는 실습 중 궁금했던 점을 스스로 질문으로 만들어 탐구해보세요.

**활용 도구 & 데이터**

자유롭게 활용

**예시**

> - **같은 질문에 대해 `gpt-4o-mini` 와 `claude-3-haiku`(Anthropic)의 답변은 어떻게 다를까?**
>     - 모델 Provider를 바꿔 같은 System Prompt·질문으로 실행해보세요. 답변 길이·어조·정확도 차이를 비교해보면 좋습니다.
> - **Temperature를 `0.0` vs `1.0` 으로 바꾸면 어떤 차이가 생길까?**
>     - 같은 질문을 5번씩 실행해, 답변이 얼마나 다양하게 나오는지 관찰해보세요. 공공 안내 챗봇에 어느 쪽이 적합한지 근거를 찾아보세요.
> - **Opening Statement 유/무에 따라 사용자 경험은 어떻게 달라질까?**
