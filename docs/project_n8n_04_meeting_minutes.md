# Project 04. 회의록 자동 정리·공유

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 회의록 자동 정리·공유 |
| 소요 시간 | 3교시 (150분) |
| 선행 Lab | **Lab 04. HTTP Request 노드**, **Lab 05. Webhook 노드**, **Lab 13. 파일 변환**, **Lab 20. Slack 알림**, **Lab 21. OpenAI 노드** |
| 사용 도구 | n8n, Slack, ChatGPT(OpenAI API), (선택) Google Drive |

`Phase 1 정의하기 (약 30분)` → `Phase 2 구현하기 (약 80분)` → `Phase 3 확장하기 (약 40분)`

### 시나리오

> 당신은 주간 부서 회의 기록을 맡은 간사입니다.
> 회의록을 한 시간 들여 정리하고, 담당자별 할 일을 따로 찾아 알려주는 일이 반복됩니다.
> 회의 녹취 텍스트(또는 Word 파일)를 Webhook으로 올리면, n8n이 GPT로 (1) 요약, (2) 결정사항, (3) 담당자별 액션아이템을 추출해 각 담당자에게 Slack DM으로 자동 공유하도록 만드세요.

### 최종 산출물

- 작동하는 n8n 워크플로우 (Webhook → 파일 텍스트화 → GPT 구조화 → 담당자 매핑 → Slack DM)
- 샘플 회의록 1건에 대해 실제 Slack DM 발송 완료
- 프롬프트와 담당자 매핑 규칙 메모

## Phase 1. 정의하기 (Define) — 약 30분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 Webhook 수신과 파일 변환을 바탕으로, 지금부터는 "회의록에서 무엇을 뽑고 누구에게 보낼지"를 스스로 정의합니다.

!!! abstract "과제"
    회의록에서 추출할 정보 카테고리 3~4개(예: 요약·결정사항·액션아이템·다음 회의)와, 각 액션아이템을 어떤 규칙으로 담당자와 매핑할지 정의하세요.

**요구사항 정리**

- [ ] 추출 카테고리 3~4개와 각 카테고리의 형식(불릿/표)이 정해졌다
- [ ] 액션아이템 필드 스키마가 정해졌다 (담당자/할 일/기한/우선순위)
- [ ] 담당자 매핑 방식이 정해졌다 (이름 → Slack user ID 매핑 표)
- [ ] 처리 대상 입력 파일 형식이 정해졌다 (txt / docx / md)
- [ ] 민감 발언·개인정보 처리 규칙이 정해졌다 ("발언자 이름 포함? 익명화?")

**활용 도구 & 데이터**

- 샘플 회의록 텍스트 (2페이지 분량) 1건 준비
- 팀원 이름 ↔ Slack user ID 매핑 표 5명 분량
- ChatGPT: 스키마 초안, 개인정보 판단 기준 질의

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    지난 회의록 한 장을 꺼내 색연필로 (1) 요약 될 문장, (2) 결정 문장, (3) 행동 문장 세 가지 색으로 표시해보세요. 색이 곧 카테고리입니다.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에게 다음과 같이 물어보세요:
    "공공기관 주간 부서 회의록에서 자동 추출할 정보 카테고리와 각 카테고리를 JSON 스키마로 표현해줘. 액션아이템에는 담당자·할 일·기한·우선순위가 포함돼야 하고, 결정되지 않은 사항은 null로 둔다."

## Phase 2. 구현하기 (Build) — 약 80분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    회의록 파일을 받아 GPT로 구조화하고, 액션아이템의 담당자 이름을 Slack user ID로 매핑해 DM을 자동 발송하는 워크플로우를 완성하세요.

**마일스톤**

☐ Webhook이 파일(또는 텍스트)을 정상 수신함을 확인했다
☐ 파일이 docx인 경우 텍스트로 변환되는 것을 확인했다 (또는 입력을 txt로 제한)
☐ OpenAI 노드가 JSON 스키마로 요약·결정사항·액션아이템을 반환한다
☐ Code(또는 Edit Fields) 노드로 액션아이템을 담당자별로 그룹화했다
☐ 담당자 이름 → Slack ID 매핑 표를 적용했다 (Sheets 또는 Code 내부 딕셔너리)
☐ Slack 노드가 담당자별 DM을 성공적으로 발송했다
☐ 요약·결정사항은 지정 채널(예: `#팀-회의`)에 일괄 게시됐다

**활용 도구 & 데이터**

- n8n: Webhook / HTTP Request / Code / OpenAI / Slack 노드
- Google Sheets: "이름 → Slack ID" 룩업 표 (선택)
- ChatGPT: 프롬프트 반복 개선, 스키마 점검

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    세 단계로 나눠 맞춰가세요.

    1. 입력 파이프라인 먼저: Webhook으로 텍스트 받기 → 출력 확인
    2. GPT 구조화 붙이기: 프롬프트 + JSON 스키마로 답변 받기
    3. Slack 출력 붙이기: 한 사람에게 DM 보내기 → 담당자 여러 명으로 확장

    중간에 "한 단계씩 수동 Execute"로 출력 포맷을 반드시 눈으로 확인하세요.

??? tip "힌트 2: 도구 활용이 막힌다면"
    담당자별 DM을 보낼 때 Slack 노드를 Loop 안에서 실행하면 각 액션아이템을 개별 발송할 수 있습니다.
    Sheets 룩업이 번거로우면 Code 노드에 간단한 딕셔너리를 두는 것도 실습용으로는 충분합니다.
    ```js
    const map = {
      "김주무관": "U012ABCD",
      "이사무관": "U034EFGH",
      "박주무관": "U056IJKL"
    };
    return items.map(i => ({
      json: { ...i.json, slack_id: map[i.json.owner] || null }
    }));
    ```

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    OpenAI 프롬프트 골격:
    ```
    너는 공공기관 회의록 정리 비서다.
    아래 회의록 본문을 읽고, 오직 JSON만 출력하라.
    {
      "summary": "3문장 이내 핵심 요약",
      "decisions": ["결정사항1", "결정사항2"],
      "action_items": [
        {"owner": "이름", "task": "할 일", "due": "YYYY-MM-DD|null", "priority": "high|mid|low"}
      ],
      "next_meeting": "YYYY-MM-DD|null"
    }
    ```

    담당자 DM 메시지 예:
    ```
    :clipboard: *주간 부서회의 액션아이템*
    • 할 일: {{task}}
    • 기한: {{due}}
    • 우선순위: {{priority}}
    원문 요약: {{summary}}
    ```

## Phase 3. 확장하기 (Extend) — 약 40분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    결과 메시지를 실제 담당자 관점에서 읽어보고, 빠졌거나 어색한 부분 2가지를 찾아 프롬프트 또는 매핑 로직을 개선하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    다음 기준으로 점검하세요.

    - 액션아이템 문장이 "동사로 시작"하는가? (예: "보고 자료 작성")
    - 기한이 없는 항목을 어떻게 표시하는가? (GPT가 임의의 날짜를 만들지 않아야 함)
    - 담당자 이름이 회의록에 명시되지 않을 때 처리 규칙이 있는가?
    - 익명/존칭 표기(주무관·사무관)가 일관된가?

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    다음 중 하나를 추가하세요. (1) Google Docs에 회의록 문서를 자동 생성·저장, (2) 액션아이템을 Notion/Asana에 카드로 생성, (3) 기한 임박(D-1) 시 자동 리마인드를 Slack으로 재발송.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    같은 "텍스트 → 구조화 → 분기 전달" 구조는 인터뷰 녹취·VOC·정책 제안서 등에 그대로 적용됩니다.
    복제해서 프롬프트만 바꾸면 다른 시나리오가 만들어집니다.
