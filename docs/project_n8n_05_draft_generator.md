# Project 05. 보도자료/공문 초안 생성기

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 보도자료/공문 초안 생성기 |
| 소요 시간 | 3교시 (150분) |
| 선행 Lab | **Lab 05. Webhook 노드**, **Lab 08. Expression 마스터**, **Lab 18. Google Sheets**, **Lab 21. OpenAI 노드** |
| 사용 도구 | n8n, Google Form, Google Sheets, ChatGPT(OpenAI API) |

📌 **진행 흐름**

`Phase 1 정의하기 (약 30분)` → `Phase 2 구현하기 (약 80분)` → `Phase 3 확장하기 (약 40분)`

> 이 프로젝트는 **Lab 05**, **Lab 08**, **Lab 18**, **Lab 21**에서 익힌 Webhook/Form 트리거, Expression, Sheets 기록, GPT 프롬프트 설계를 통합 적용합니다.

### 시나리오

> 당신은 홍보·총무 담당 주무관입니다.
> 매주 부서·사업에서 보도자료와 공문 초안 요청이 쏟아지지만, 매번 "양식은 같은데 내용만 다른" 글을 처음부터 쓰느라 시간이 듭니다.
> Google Form으로 핵심 정보를 접수하면, n8n이 양식(기관명·제목·리드·본문·담당자)에 맞춰 GPT가 초안을 써내고, Google Sheets에 접수 이력과 초안을 남기도록 만드세요. 초안은 "검토 완료 전"임을 명시해야 합니다.

### 최종 산출물

- 작동하는 n8n 워크플로우 (Form Webhook → 입력 검증 → GPT 초안 → Sheets 기록)
- 2가지 이상 양식 템플릿 (예: 보도자료 / 대내 공문)
- Google Sheets 이력 시트 (접수 시각, 요청자, 양식, 초안 텍스트, 상태)

## Phase 1. 정의하기 (Define) — 약 30분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 Webhook 수신과 Expression 활용을 바탕으로, 지금부터는 "양식과 입력 스키마"를 스스로 정의합니다.

!!! abstract "과제"
    작성할 양식 2종(예: 보도자료, 대내 공문)을 골라, 각 양식의 구조와 반드시 받아야 할 입력 필드 5~7개를 정의하세요.

**요구사항 정리**

- [ ] 대상 양식 2종이 선정되었다
- [ ] 각 양식의 고정 골격(섹션 제목·순서)이 정리되었다
- [ ] 필수 입력 필드 5~7개와 형식이 정의되었다 (일자, 기관명, 제목, 리드, 본문 개요, 담당자 등)
- [ ] 입력이 누락되었을 때의 기본값 또는 검증 규칙이 정해졌다
- [ ] 초안 말미에 "검토 전 초안" 면책 문구가 들어가도록 규칙이 정해졌다

**활용 도구 & 데이터**

- Google Form 초안 (필드 구성만, 실제 발행은 선택)
- 실제 보도자료/공문 샘플 2~3건 (어조·구조 분석용)
- ChatGPT: 양식 골격 정리, 필드 스키마 제안

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    부서 홈페이지나 과거 문서함에서 최근 보도자료·공문 2~3건을 꺼내 공통점만 발췌하세요.
    "항상 들어가는 문장"과 "매번 바뀌는 문장"을 색으로 구분하면, 고정 템플릿과 변수 필드가 자연스럽게 나옵니다.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에게 다음과 같이 물어보세요:
    "공공기관 보도자료 표준 구조(헤더·리드·본문·문의처)와, 각 섹션을 생성하기 위해 필요한 최소 입력 필드를 JSON 스키마로 정리해줘."

    결과를 우리 부서 실제 보도자료와 대조해 조정하세요.

## Phase 2. 구현하기 (Build) — 약 80분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    Google Form(또는 Webhook 직접 호출)으로 입력을 받아, 선택한 양식에 맞게 초안 텍스트를 생성하고 Sheets 이력에 기록하는 워크플로우를 완성하세요.

**마일스톤**

☐ Webhook(또는 Form 트리거)로 양식 선택값과 필드를 정상 수신했다
☐ IF/Switch로 양식별 프롬프트가 분기된다
☐ OpenAI 노드가 양식 골격을 지킨 초안을 생성한다
☐ 초안 끝에 "본 자료는 검토 전 초안입니다" 면책 문구가 자동 삽입된다
☐ Google Sheets에 접수 시각·요청자·양식·초안·상태(`초안 완료`)가 기록됐다
☐ 담당자에게 초안 링크(또는 본문)가 메일/Slack으로 전달된다 (선택)

**활용 도구 & 데이터**

- n8n: Webhook / IF or Switch / OpenAI / Google Sheets / Gmail(선택)
- Google Form & Sheets 연결 (응답 자동 저장)
- ChatGPT: 양식별 프롬프트 초안 작성

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    한 양식(예: 보도자료)만 먼저 성공시키세요. 그 다음 Switch 노드로 두 번째 양식을 추가하면 전체 구조가 잡힙니다.
    초기 프롬프트는 짧게 "제목/리드/본문 3단락" 정도로 시작해, 결과를 보면서 조건을 덧붙이는 방식이 빠릅니다.

??? tip "힌트 2: 도구 활용이 막힌다면"
    양식 골격을 프롬프트에 명시적으로 넣으세요.
    ```
    아래 구조로만 작성한다.
    [헤더]
    - 기관명 / 일자 / 담당자
    [제목]
    - 한 문장, 40자 이내
    [리드]
    - 2~3문장, 누가·무엇을·왜를 포함
    [본문]
    - 3단락, 각 단락 3문장 이내
    [문의처]
    - 담당자 / 연락처
    ```

    Expression으로 Form 응답값을 프롬프트에 삽입: `제목 힌트: {{ $json.title_hint }}`

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    Sheets 컬럼 구성 예:
    | 접수일시 | 요청자 | 양식 | 입력 요약 | 초안 본문 | 상태 | 검토자 |

    상태는 `초안 완료` → `검토 요청` → `승인` → `발행` 단계로 관리하면, 나중에 승인 플로우로 확장할 때 편합니다.

## Phase 3. 확장하기 (Extend) — 약 40분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    생성된 초안 3건을 읽고 "우리 기관 어조와 맞지 않는 부분" 2곳을 찾아 프롬프트를 수정하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    프롬프트에 "어조 규칙"을 구체 문장으로 제시하세요.

    - 수동태 금지, 능동태로 작성
    - "~할 예정이다" 대신 "~한다" 현재형
    - 외래어는 괄호 병기(예: "플랫폼(Platform)")
    - 문장 길이 평균 40자 이내

    어조 규칙을 프롬프트 상단 `너는 ~다` 옆에 고정시키면 결과 일관성이 크게 좋아집니다.

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    다음 중 하나를 추가하세요. (1) 초안 완료 시 검토자(상급자)에게 자동 메일 발송, (2) 승인 버튼(체크박스) 클릭 시 상태 갱신 및 발행 채널로 재발송, (3) 과거 초안을 검색해 유사도 높은 건을 참고로 프롬프트에 넣기(간단 RAG).

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    이 워크플로우는 "입력 → 템플릿 → 생성 → 이력" 네 단계의 일반 패턴입니다.
    같은 뼈대로 "보도자료 → 카드뉴스 문구", "공문 → 결재 요지서"도 복제해 만들 수 있습니다. 양식과 프롬프트만 바꾸면 됩니다.
