### 시나리오

> 당신은 민원 처리 현황을 관리하는 주무관입니다.
> 법정 처리 기한이 가까워진 건을 뒤늦게 알게 되어 야근으로 몰아 처리하는 일이 반복됩니다.
> Sheets 민원 대장을 매일 오전 9시에 자동 점검해, 접수일 기준 D-3·D-1 건을 담당자 Gmail로 리마인드하고, 이미 기한을 넘긴 지연 건은 Slack으로 상급자에게 즉시 보고하는 워크플로우를 만드세요.

### 최종 산출물

- 작동하는 n8n 워크플로우 (Schedule → Sheets 읽기 → Loop → 날짜 계산 → 분기 알림)
- 알림 샘플 (Gmail 리마인드 1건, Slack 지연 보고 1건)
- 기한 계산 Expression과 알림 정책 메모

## Phase 1. 정의하기 (Define) — 약 30분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 Schedule과 날짜 Expression을 바탕으로, 지금부터는 "기한 정책과 알림 규칙"을 스스로 정의합니다.

!!! abstract "과제"
    민원 대장에서 처리 기한을 어떻게 계산할지(접수일 + N일 등) 정의하고, D-3 / D-1 / D+(지연) 각 상태별 알림 수단·수신자·메시지 포맷을 표로 정리하세요.

**요구사항 정리**

- [ ] 기한 계산 규칙이 정의되었다 (예: 접수일 + 7영업일)
- [ ] 알림 발송 시점(시간)과 주기가 정해졌다 (매일 09:00)
- [ ] D-3 / D-1 / D+ 각 상태의 수신자·수단이 정해졌다
- [ ] "이미 처리 완료" 건은 제외하는 규칙이 정해졌다
- [ ] 주말·공휴일 취급 규칙이 정해졌다 (단순화해 영업일만 고려해도 됨)

**활용 도구 & 데이터**

- Google Sheets 민원 대장 (접수일, 기한, 담당자, 상태 컬럼 포함, 10건 이상)
- ChatGPT: 날짜 Expression 문법 질의
- 담당자 이메일 매핑 표

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    "알림이 울리지 않아 사고가 났던 과거 사례"를 1~2개 떠올려보세요. 그 상황을 어떻게 먼저 포착했어야 했는지가 곧 정책입니다.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에게 다음과 같이 물어보세요:
    "공공기관 민원 처리 법정 기한 관리 관점에서 D-3·D-1·초과 상태별로 적절한 알림 수신자·수단·메시지 톤을 표로 제안해줘. 영업일 계산 방식도 함께."

## Phase 2. 구현하기 (Build) — 약 80분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    Schedule로 매일 09:00 실행되는 워크플로우를 만드세요. Sheets에서 미처리 민원을 읽어 건별로 잔여일수를 계산한 뒤, 분기별로 Gmail 리마인드 혹은 Slack 상급자 보고를 자동 발송합니다.

**마일스톤**

☐ Schedule Trigger가 매일 09:00에 실행되도록 설정됐다
☐ Google Sheets 노드로 `상태 != 완료` 건만 필터링해 가져왔다
☐ Loop 또는 Split In Batches로 각 건을 개별 처리하는 흐름을 구성했다
☐ Expression으로 `잔여일수 = 기한 - 오늘` 계산이 정확하다
☐ IF/Switch로 D-3 / D-1 / 지연 세 분기가 동작한다
☐ D-3·D-1은 담당자 Gmail로 개별 발송, 지연 건은 상급자 Slack에 묶어 보고됐다
☐ 테스트 대장 10건 중 알림이 정확한 수만큼(미리 계산한 기대값) 발송됐다

**활용 도구 & 데이터**

- n8n: Schedule / Google Sheets / Loop(Split In Batches) / IF or Switch / Gmail / Slack
- Sheets 민원 대장 (10건 이상, 다양한 기한 포함)
- ChatGPT: 날짜 함수·Expression 점검

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    먼저 Schedule 없이 수동 실행으로 전체 흐름을 검증하세요.

    1. Sheets 읽기 → 행 수 확인
    2. Expression으로 잔여일수 컬럼 추가 → 한 행만 눈으로 검증
    3. IF 분기 후 각 분기 끝에 Slack "테스트 메시지"만 보내 각 케이스가 흐르는지 확인
    4. 메시지를 실제 포맷으로 교체
    5. 마지막에 Manual을 Schedule로 바꿉니다

??? tip "힌트 2: 도구 활용이 막힌다면"
    날짜 차이 Expression 예:
    ```
    {{ Math.floor( (new Date($json.deadline) - new Date()) / (1000*60*60*24) ) }}
    ```

    또는 n8n의 DateTime 함수 사용:
    ```
    {{ $json.deadline.toDateTime().diff($now, 'days').days }}
    ```

    Loop 대신 전체 리스트를 한 번에 Filter → Merge로 합치는 방법도 있습니다. 규모가 작으면 Loop가 이해하기 쉽습니다.

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    Gmail 리마인드 본문 예:
    ```
    [민원 처리 리마인드 · D-{{days_left}}]
    - 접수번호: {{id}}
    - 제목: {{title}}
    - 민원인: {{applicant}}
    - 기한: {{deadline}}
    - 상태: {{status}}

    남은 처리 기한이 {{days_left}}일입니다. 오늘 중 현황을 확인해주세요.
    ```

    Slack 지연 보고(집계):
    ```
    :rotating_light: *기한 초과 민원 {{count}}건*
    • {{id1}} — {{title1}} (담당: {{owner1}}, 초과 {{over1}}일)
    • {{id2}} — ...
    ```

## Phase 3. 확장하기 (Extend) — 약 40분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    며칠에 걸쳐 실행해보고 "필요 없는 알림이 자주 울리는가"와 "놓친 케이스가 있는가"를 각각 한 가지씩 찾아 정책을 수정하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    다음 패턴을 점검하세요.

    - 동일 민원에 대해 D-3·D-1·D+가 연달아 울려 담당자가 피로한가? (→ 상태 변경이 있을 때만 재발송)
    - 주말에 보낸 알림이 월요일에 과도하게 쌓이는가? (→ 영업일 기준으로만 실행)
    - "상태 == 완료"인데 알림이 나갔는가? (→ Sheets 필터 검증)
    - 담당자 부재 시 대체 수신자가 있는가? (→ 매핑 표에 백업 담당자 컬럼 추가)

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    다음 중 하나를 추가하세요. (1) 주간 지연 통계 리포트 자동 발송(지연 추이 그래프용 데이터 Sheets 기록), (2) Google Calendar에 각 민원의 기한을 자동 등록, (3) Gmail 답장 확인(완료 회신) 시 Sheets 상태를 `완료`로 자동 업데이트.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    같은 "데이터 + 날짜 + 알림" 구조는 결재 지연·계약 만료·재물조사 기한 등에 그대로 재활용됩니다.
    대상 Sheet와 기한 계산 규칙만 바꾸면 같은 봇이 다른 업무를 관리합니다.
