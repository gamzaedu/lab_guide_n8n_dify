# Project 03. 내부 규정/지침 검색 챗봇

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 내부 규정/지침 검색 챗봇 |
| 소요 시간 | 2교시 (100분) |
| 선행 Lab | **Lab 04 · 지식베이스 만들기**, **Lab 05 · 내부 규정 QA 봇**, **Lab 06 · 검색 설정 최적화** |
| 사용 도구 | Dify Cloud (Chatbot + Knowledge), ChatGPT, 규정집 PDF |

📌 **진행 흐름**

`Phase 1 정의하기 (약 20분)` → `Phase 2 구현하기 (약 55분)` → `Phase 3 확장하기 (약 25분)`

> 이 프로젝트는 **Lab 04 · 지식베이스 만들기**, **Lab 05 · 내부 규정 QA 봇**, **Lab 06 · 검색 설정 최적화**에서 익힌 지식베이스 구축·챗봇 연결·검색 파라미터 튜닝을 통합 적용합니다.

### 시나리오

> 당신은 부서 총무·감사 담당입니다.
> 신규 직원에게서 복무규정(연차·근태), 출장지침(교통비·숙박비), 물품관리(구매 절차) 관련 질문이 매일 반복적으로 들어오고, 같은 답을 여러 번 설명하는 데 시간이 많이 들어갑니다. 당신은 부서 규정집 PDF 2~3건을 기반으로 질문에 **근거 조항과 원문 스니펫을 함께** 답하는 챗봇을 만들어 부서 공용 링크로 배포하려 합니다.

### 최종 산출물

- 규정집 PDF 2~3건이 업로드된 Dify 지식베이스
- 지식베이스와 연결된 Chatbot 앱
- 테스트 질문 10개 중 7건 이상에서 올바른 근거 조항을 인용한 응답
- 정답률 측정 결과 (간단한 표)

## Phase 1. 정의하기 (Define) — 약 20분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 지식베이스 업로드와 검색 설정을 바탕으로, 지금부터는 스스로 문제를 정의합니다.

!!! abstract "과제"
    챗봇이 다룰 규정 범위, 주요 질문 목록, 답변 표준 형식, 모르는 질문 처리 정책을 정리하세요.

**요구사항 정리**

- [ ] 포함 범위가 확정되었다 (규정집 문서 2~3개 선정, 샘플 PDF 준비)
- [ ] 주요 질문 10개가 수집되었다 (연차·출장·경비·물품신청·근태 등)
- [ ] 답변 표준 형식이 정의되었다 (요약 2줄 + 근거 조항 번호 + 원문 인용)
- [ ] 모르는 질문 처리 정책이 정해졌다 ("규정집에서 찾지 못했습니다" + 담당자 연락처 안내)

**활용 도구 & 데이터**

- Dify Knowledge (지식베이스) + Chatbot
- 규정집 PDF 2~3건 (복무규정 · 출장지침 · 물품관리지침 등)
- ChatGPT: 주요 질문 브레인스토밍, 답변 형식 다듬기

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    "최근 한 달간 직원들에게서 실제로 받은 질문"을 10개 떠올려 적으세요. 기억이 안 나면 신규 직원 1명에게 "최근에 헷갈려서 물어봤던 것" 5개만 물어도 질문 목록의 절반이 채워집니다.

    질문 옆에 "이 답을 찾으려면 어떤 규정집의 몇 조를 봐야 하나?"를 메모해두세요. 나중에 테스트 정답지가 됩니다.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    답변 형식을 다음 3요소로 고정하면 검수가 쉬워집니다:

    ```
    [요약] 2줄 이내, 질문에 직접 답
    [근거] 규정집명 · 조문 번호
    [원문] 해당 조문 한 문단 인용
    ```

    "근거와 원문 없이 요약만 나오면 안 된다"를 명시적 제약으로 프롬프트에 넣을 예정이라는 것을 미리 요구사항에 적어두세요.

## Phase 2. 구현하기 (Build) — 약 55분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    Dify 지식베이스에 규정집을 업로드하고 챗봇과 연결한 뒤, Phase 1에서 준비한 질문 10개로 정답률을 측정하세요.

**마일스톤**

☐ 지식베이스를 생성하고 PDF 2~3개를 업로드했다
☐ 청킹(chunk size / overlap)을 한글 규정 문서에 맞게 조정했다
☐ Chatbot 앱에 지식베이스를 연결하고 컨텍스트 Top-K·score 임계값을 설정했다
☐ 시스템 프롬프트에 답변 표준 형식 + 환각 방지 + 모르는 질문 처리 규칙을 작성했다
☐ 질문 10개를 순차 테스트하고 정답/오답/부분정답을 표로 기록했다 (7건 이상 정답 목표)

**활용 도구 & 데이터**

- Dify Knowledge: Embedding Model, Chunk Settings, Retrieval Settings
- Dify Chatbot: Context 연결, System Prompt, Citations 설정
- ChatGPT: 시스템 프롬프트 리뷰, 테스트 결과 분석 보조

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    전체를 한 번에 세팅하려 하지 말고 단계별로 확인하세요:

    1. **지식베이스만 먼저**: PDF 1개를 올리고 Dify의 Test 기능(또는 Retrieval Test)으로 "연차는 며칠 쓸 수 있나요?" 같은 질문으로 관련 청크가 뽑히는지 확인
    2. **나머지 PDF 추가**: 모든 문서가 각각 검색되는지 확인
    3. **챗봇 연결**: 가장 간단한 프롬프트로 붙여서 검색 결과가 응답에 들어가는지 확인
    4. **프롬프트 정교화**: 답변 형식, 거절 규칙 추가

    각 단계마다 같은 테스트 질문 1개로 결과를 비교하세요.

??? tip "힌트 2: 도구 활용이 막힌다면"
    한글 규정 문서에서 **청크가 조문 중간을 자르면** 답변 품질이 떨어집니다. Lab 06에서 배운 대로 다음을 시도하세요:

    - chunk size를 500~800자로 조정
    - overlap을 100~150자 정도로 늘려 조문 경계 손실 방지
    - 필요하면 "제N조"를 기준으로 한 수동 splitter 사용 (Dify의 Custom Segmentation)

    Retrieval Top-K는 3~5가 적당합니다. 너무 크면 환각, 너무 작으면 누락이 생깁니다.

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    시스템 프롬프트 뼈대 예시:

    ```
    당신은 내부 규정 안내 담당입니다. 제공된 Context에 근거해서만 답하세요.

    [응답 형식]
    [요약] (2줄 이내)
    [근거] (규정집명 · 조문번호)
    [원문] (해당 조문 직접 인용)

    [규칙]
    - Context에 답이 없으면 추측하지 말고 정확히 다음 문장으로 답할 것:
      "규정집에서 해당 내용을 찾지 못했습니다. 총무팀(내선 000)으로 문의 바랍니다."
    - [요약]에는 Context에 없는 사실을 추가하지 말 것.
    ```

    테스트 결과 기록표 예시:

    | 질문 | 기대 근거 | 실제 근거 | 정답 여부 |
    |------|-----------|-----------|----------|
    | 연차는 며칠? | 복무규정 §15 | 복무규정 §15 | ✅ |

## Phase 3. 확장하기 (Extend) — 약 25분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    정답률이 70% 미만이거나 환각(Context에 없는 내용 삽입)이 발생한 사례 3건을 뽑아, 청킹·검색 설정 또는 프롬프트를 수정한 뒤 재측정하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    실패 유형별로 처방이 다릅니다:

    - **관련 조문이 검색되지 않음** → chunk size/overlap 조정, 파일 재업로드
    - **검색은 되는데 답변에 반영 안 됨** → Top-K 증가, 또는 프롬프트에 "Context를 반드시 인용" 강조
    - **환각(Context에 없는 내용)** → score 임계값 상향, 프롬프트에 "Context에 없으면 거절" 강화

    한 번에 여러 변수를 바꾸지 말고, 한 가지 변경 후 같은 10개 질문을 돌려 효과를 측정하세요.

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    작동하는 챗봇에 다음 중 하나를 추가해보세요: (1) 다른 부서 규정 PDF 3건을 추가 업로드해 "부서 필터" 변수로 답변 범위 제한, (2) n8n 연계로 슬랙 슬래시 명령(`/규정 연차`)에서 호출, (3) 응답에 "관련 양식 다운로드 링크"를 함께 표시.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    가장 쉬운 확장은 **부서별 규정 분리**입니다. 지식베이스를 부서별로 나눠 만든 뒤, 입력 변수 `부서`로 어느 지식베이스를 사용할지 분기하면 됩니다.

    슬랙 연계는 Lab 14(n8n에서 Dify API 호출)를 참고해, n8n의 Slack Trigger → HTTP Request(Dify API) → Slack 응답 흐름으로 구성할 수 있습니다.
