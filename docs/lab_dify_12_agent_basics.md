!!! abstract "과제"
    **Agent 앱**을 생성해 **Current Time** 과 **Calculator** 2개 내장 도구를 연결하고, 이 도구들을 혼합해야 답이 나오는 질문 3개를 던져 Agent가 스스로 어떤 도구를 언제 부르는지 관찰하세요.

**활용 도구 & 데이터**

- **도구:** Dify Agent 앱 (이름: `Lab 12 - 업무 보조 Agent`)
- **연결할 내장 도구 2종:**

| 도구 | 용도 |
|------|------|
| Current Time | 현재 날짜·시간 조회 |
| Calculator | 수학 계산 (사칙연산·제곱 등) |

- **System Prompt (Instructions):**

```
당신은 공무원 업무 보조 Agent입니다.
필요할 때 도구를 자동으로 호출해 정확한 답을 제공합니다.

원칙:
- 오늘 날짜·시간이 필요한 질문은 반드시 Current Time 도구를 호출합니다.
- 계산이 필요한 질문은 반드시 Calculator 도구를 호출합니다.
- 도구 결과를 본 뒤, 사용자가 이해하기 쉬운 한국어 문장으로 답변합니다.
```

- **테스트 질문 3종:**
    1. "오늘 날짜부터 30일 뒤 날짜가 며칠이지?"
    2. "시급 11,500원으로 하루 8시간, 주 5일, 한 달(4주) 일하면 월급이 얼마?"
    3. "오늘은 며칠이고, 지금 시각은 몇 시야?"

**제작 순서:**

1. Studio → **Create App** → **Agent** 선택 → 이름 `Lab 12 - 업무 보조 Agent`
2. 좌측 **Tools** 영역 → **[+ Add]** → Built-in Tools 중 **Current Time** 선택 → Authorize
3. 동일하게 **Calculator** 추가
4. Instructions에 위 System Prompt 복사
5. 모델: `gpt-4o-mini` / Reasoning Mode: **Function Calling** (지원 모델에 자동 활성)
6. Max Iterations: `5` (무한 루프 방지)
7. Debug & Preview에서 3개 질문 실행 → **Tools Used** 패널에서 호출된 도구 확인

??? note "예시"

    > **Q: 오늘 날짜부터 30일 뒤 날짜가 며칠이지?**
    >
    > **Agent 동작(Run History):**
    >
    > ```
    > [Step 1] Tool call: Current Time → "2026-04-19 (Sunday)"
    > [Step 2] Tool call: Calculator → "2026-04-19 + 30 days"
    >           (또는 Calculator가 직접 날짜 연산 못하면 LLM이 일수 계산: 19 + 30 = 49)
    > [Step 3] 최종 답변 생성
    > ```
    >
    > **최종 응답:** "오늘 2026-04-19 기준 30일 뒤는 2026-05-19(월요일)입니다."
    >
    > **Q: 시급 11,500원 × 8시간 × 5일 × 4주 월급은?**
    >
    > **Agent 동작:**
    >
    > ```
    > [Step 1] Tool call: Calculator("11500 * 8 * 5 * 4") → 1,840,000
    > [Step 2] 최종 답변
    > ```
    >
    > **응답:** "월급은 1,840,000원입니다. (시급 11,500원 × 일 8시간 × 주 5일 × 4주 기준)"
    >
    > **확인 포인트:**
    >
    > - Tools Used 패널에 호출된 도구가 정확히 표시되는가
    > - 계산 질문에서 Calculator를 호출했는가 (LLM이 머릿속으로 푸는 것이 아니라)
    > - 도구 결과를 사용자 친화적 한국어로 최종 변환했는가

### Step 2. 분석과 성찰 (Analyzing & Reflecting)

??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"
        객관적인 피드백을 구하되, 판단은 스스로 합니다.

    **[의미]**

    Step 1 에서 만든 결과물을 한 발 물러서서 객관적으로 바라보는 단계입니다.
    혼자서는 보이지 않던 것들이, 다른 시선을 통해 드러납니다.

    **[지침]**

    - 1단계 작업물에 대해, 객관적인 피드백을 얻습니다
    - 다른 탐색하기 결과물을 확인합니다
        - 다른 동료의 작업 결과물 | AI 의 생성 결과물 | 웹검색 등
    - AI 를 사용할 땐 반드시 비판적인 자세를 유지합니다.
        - AI 가 생성한 결과물 및 지침은 그대로 받아들이지 않고, 의심합니다.

!!! abstract "과제"
    같은 3개 질문을 **(A) Function Calling** 모드와 **(B) ReAct** 모드로 각각 실행해 호출 정확도·응답 속도·실패 케이스를 비교하세요. 도구를 **쓰지 말아야 할 질문**(예: "안녕하세요")도 포함해 불필요 호출이 있는지 관찰합니다.

**활용 도구 & 데이터**

- **도구:** Step 1 Agent에서 Reasoning Mode만 변경
- **비교 설계:**

| 버전 | Reasoning Mode |
|------|----------------|
| A | Function Calling |
| B | ReAct |

- **테스트 질문 확장 (5건):**
    1. 오늘부터 30일 뒤?
    2. 시급 월급 계산
    3. 오늘 날짜 + 시각
    4. "안녕하세요" (도구 불필요)
    5. "파리 에펠탑이 몇 m야?" (도구 불가, LLM 상식 or 모름)

**측정 항목:**

| 항목 | 측정 |
|------|------|
| 필요한 도구를 부른 횟수 | 3건 중 (질문 1~3) |
| 불필요하게 도구를 부른 횟수 | 2건 중 (질문 4~5) |
| 평균 응답 시간 | 초 |
| 최종 답 정확도 | 5건 중 ✅/× |

**비교 방법:**

1. A/B 모드에서 각 질문 실행 후 Run History 분석
2. Tools Used 수를 집계해 호출 정확도 계산

**예시**

> **비교 분석표 — A(Function Calling) vs B(ReAct)**
>
> | 항목 | A | B |
> |------|---|---|
> | 필요한 도구 호출 | 3/3 | 3/3 |
> | 불필요 호출 | 0/2 | 1/2 (Q4 "안녕"에 Current Time 호출) |
> | 평균 응답 시간 | 2.1초 | 3.4초 |
> | 최종 정확도 | 5/5 | 4/5 (Q4 답 이상) |
>
> **분석 결과:**
>
> - Function Calling 지원 모델에서는 A가 **더 빠르고 정확**, 도구를 필요한 때만 부름
> - ReAct는 도구 없는 모델(오픈소스 로컬 LLM 등)에서 대안으로 유용하지만 **불필요 호출 확률 증가**
> - 실무 기본: 모델이 Function Calling을 지원하면 A 사용, 아니면 B
> - System Prompt에 "인사 등 도구가 불필요한 경우 바로 응답" 같은 가이드를 추가하면 B의 오작동을 줄일 수 있음

### Step 3. 창조하기 (Creating)

??? note "해당 단계 사용법 살펴보기"

    !!! tip "원칙"
        완성품이 아니라, 작동하는 시작품을 만듭니다.

    **[의미]**

    앞서 얻은 통찰을 바탕으로, 나만의 결과물을 만들어 봅니다.

    **[지침]**

    - 2단계에서 얻은 피드백을 원하는 만큼 반영합니다.
        - 중요한 점은, 받아들일 수 있는 만큼만 받아들이는 것입니다.
    - 내 상황과 스타일에 맞게 최적화된 결과물을 만들어 봅니다.
    - 처음부터 완성하는 것이 아닌 계속해서 개선해 나가는 것입니다.
        - 가장 작은 단위부터 프로토타입으로 만들어 보세요

!!! abstract "과제"
    본인 업무에 도움이 될 **"업무 보조 Agent"** 를 완성하세요. 도구 **3개 이상**(예: Current Time + Calculator + Web Search)을 연결하고, 각 도구를 언제 써야 하는지 System Prompt에 명시합니다. 실제 업무 질문 5건으로 테스트하고 Publish합니다.

**활용 도구 & 데이터**

- **도구:** Dify Agent
- **완성 체크리스트:**

| 항목 | 기준 |
|------|------|
| 내장 도구 | 3개 이상 |
| System Prompt | 도구 사용 조건 명시 |
| Max Iterations | 5 이하 |
| 테스트 | 실제 업무 5건 |
| 정확도 | 5/5 도구 호출 정확 |

**완성 순서:**

1. 업무 정리: 어떤 업무에서 "실시간 정보·계산·검색"이 필요한가
2. 도구 3개 선정 후 연결
3. System Prompt에 "A 유형 질문은 X 도구, B 유형은 Y 도구" 명시
4. 실제 질문 5건 실행 → 도구 호출 정확도 측정
5. Max Iterations = 5 제한, 무한 루프 방지
6. Publish → 동료와 공유

**예시**

> **완성 Agent: "공보실 업무 보조 Agent"**
>
> **연결 도구:**
>
> | 도구 | 용도 |
> |------|------|
> | Current Time | 오늘 날짜·요일 |
> | Calculator | 수량·예산 계산 |
> | Google Search (또는 DuckDuckGo) | 최신 뉴스·정부 보도자료 검색 |
>
> **System Prompt (발췌):**
>
> ```
> 당신은 공보실의 업무 보조 Agent입니다. 아래 규칙대로 도구를 사용하세요.
> - 날짜/요일/오늘 기준: Current Time
> - 예산 추정·배포 부수 계산: Calculator
> - "요즘 ○○ 이슈" 같은 최신 정보: Google Search
> - 인사·일반 대화는 도구를 호출하지 마세요.
> ```
>
> **테스트 결과 (5/5 통과):**
>
> | # | 질문 | 호출 도구 | 답 정확도 |
> |---|------|-----------|----------|
> | 1 | 오늘이 무슨 요일이지? | Current Time | ✅ |
> | 2 | 전단지 2만부 × 단가 120원이면 예산? | Calculator | ✅ |
> | 3 | 최근 청년 정책 관련 보도자료? | Google Search | ✅ |
> | 4 | 보도자료 초안 써줘 (일반 주제) | (도구 X) | ✅ |
> | 5 | 오늘부터 7일 뒤 예산 결재가능? | Current Time + Calculator | ✅ |
