# Project 02. 공문 메일 자동 분류·요약

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 공문 메일 자동 분류·요약 |
| 소요 시간 | 2교시 (100분) |
| 선행 Lab | **Lab 02. Trigger 노드 비교**, **Lab 06. IF 노드**, **Lab 08. Expression 마스터**, **Lab 19. Gmail 노드**, **Lab 21. OpenAI 노드** |
| 사용 도구 | n8n, Gmail, ChatGPT(OpenAI API), Slack(선택) |

📌 **진행 흐름**

`Phase 1 정의하기 (약 20분)` → `Phase 2 구현하기 (약 55분)` → `Phase 3 확장하기 (약 25분)`

### 시나리오

> 당신은 부서의 대표 공용 메일함을 매일 관리하는 담당자입니다.
> 하루 수십 통의 메일이 섞여 들어옵니다. 공문, 민원, 광고, 내부 공지가 뒤엉켜 있어 중요한 공문을 놓칠 때가 있습니다.
> Gmail에서 수신함을 주기적으로 확인해 (1) 규칙 기반으로 라벨을 자동 분류하고, (2) "공문·긴급" 카테고리만 GPT가 3줄 요약해 Slack(또는 본인 메일)으로 보내주는 n8n 워크플로우를 만드세요.

### 최종 산출물

- 작동하는 n8n 워크플로우 (Schedule → Gmail 검색 → IF 분류 → GPT 요약 → Slack/Gmail 전달)
- 분류 규칙과 요약 프롬프트 메모
- 테스트 메일 3건 이상에 대해 분류·요약이 의도대로 동작한 스크린샷

## Phase 1. 정의하기 (Define) — 약 20분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 IF 분기와 Expression을 바탕으로, 지금부터는 스스로 분류 규칙을 정의합니다.

!!! abstract "과제"
    우리 부서 메일함에 들어오는 메일을 3~5개 카테고리로 나누는 규칙을 정의하세요. 각 카테고리마다 "무엇을 기준으로", "어떤 키워드/발신자로" 판별할지 구체적으로 적습니다.

**요구사항 정리**

- [ ] 카테고리 3~5개가 정의되었다 (예: 공문 / 민원 / 내부 공지 / 광고 / 기타)
- [ ] 각 카테고리의 판별 규칙(키워드, 발신 도메인)이 작성되었다
- [ ] "요약이 필요한 카테고리"와 "라벨만 붙일 카테고리"가 구분되었다
- [ ] 요약의 독자(본인? 팀장? Slack 채널?)와 전달 방식이 정해졌다

**활용 도구 & 데이터**

- Gmail 테스트 라벨: `테스트/미분류` 라벨에 샘플 메일 몇 건을 미리 넣어두면 편합니다
- ChatGPT: 키워드 브레인스토밍, 분류 기준 점검

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    지난 일주일 수신함을 스크롤하며 메일을 손으로 분류해보세요. 손으로 나눈 기준이 곧 자동화 규칙의 초안입니다.
    "왜 이 메일을 이 그룹에 넣었는가?"를 한 줄로 설명할 수 있어야 규칙으로 바뀝니다.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에게 다음과 같이 물어보세요:
    "공공기관 부서 공용 메일함에서 자주 오는 메일 유형과, 각 유형을 구분할 수 있는 발신자 도메인·제목 키워드·본문 패턴을 표로 정리해줘."

    AI가 준 기준을 우리 부서 실제 수신함과 대조해 조정하세요.

## Phase 2. 구현하기 (Build) — 약 55분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    주기적으로 Gmail 수신함을 검사하고, 제목·발신자 규칙으로 카테고리를 판별해 라벨을 부여하세요. "공문·긴급" 카테고리에 해당하면 GPT로 3줄 요약한 뒤 Slack(또는 본인 메일)으로 보내는 워크플로우를 완성합니다.

**마일스톤**

☐ Schedule Trigger로 15분 간격 실행이 설정됐다
☐ Gmail 노드가 지정 라벨(또는 특정 기간)의 메일을 읽어오는 것을 확인했다
☐ IF 노드로 제목/발신자 기준 분기가 동작한다 (테스트 메일 2건 이상 검증)
☐ 공문 카테고리에 해당할 때 OpenAI 노드가 3줄 요약을 반환한다
☐ 요약 결과가 Slack(또는 본인 메일)으로 전달되었다
☐ Gmail 라벨이 자동으로 붙은 것을 수신함에서 확인했다

**활용 도구 & 데이터**

- n8n: Schedule Trigger / Gmail / IF / OpenAI / Slack(또는 Gmail) 노드
- Gmail: 테스트 라벨에 샘플 메일 3~5건 미리 준비
- ChatGPT: 3줄 요약 프롬프트 설계

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    Gmail 전체 수신함을 한 번에 건드리지 말고, 전용 테스트 라벨(`테스트/미분류`)로 범위를 좁혀 시작하세요.
    순서는 "메일 읽기 → 한 건 분기 → 한 건 요약 → 한 건 전송"으로, 가장 좁은 케이스부터 끝까지 연결한 뒤 규칙을 늘려갑니다.

??? tip "힌트 2: 도구 활용이 막힌다면"
    IF 조건 예시
    - 제목에 `공문` 또는 `알림` 포함 → 공문
    - 발신자 도메인이 `@korea.kr`, `@go.kr` → 공문
    - 제목에 `민원` → 민원
    - 제목에 `광고`, `구독` → 광고

    여러 조건을 조합할 때는 Switch 노드(Lab 07)가 더 깔끔할 수 있습니다. IF로 익숙해진 뒤 Switch로 리팩토링해도 좋습니다.

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    3줄 요약 프롬프트 예시:
    ```
    너는 공공기관 주무관의 메일 비서다.
    아래 메일을 한국어로 정확히 3줄로 요약해라.
    1줄: 핵심 사안 한 문장
    2줄: 요청/기한(있다면 날짜와 함께)
    3줄: 내가 취해야 할 행동 한 가지

    메일 제목: {{ $json.subject }}
    발신자: {{ $json.from }}
    본문: {{ $json.snippet }}
    ```

    Slack 전송 포맷 예:
    ```
    :mail: *[공문]* {{제목}}
    {{요약 3줄}}
    🔗 {{gmail 링크}}
    ```

## Phase 3. 확장하기 (Extend) — 약 25분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    테스트 메일 10건을 돌려보고 분류 정확도와 요약 품질을 점검하세요. 오분류 2건을 골라 원인을 분석하고, 규칙이나 프롬프트를 수정해 개선하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    오분류의 원인을 다음 4가지 중 어디에 해당하는지 따져보세요.

    - 키워드가 너무 좁다 (→ 단어 추가)
    - 키워드가 너무 넓다 (→ 예외 조건 추가)
    - 발신자 도메인 기준이 빠졌다 (→ IF에 조건 추가)
    - GPT 요약이 3줄 규칙을 어긴다 (→ 프롬프트에 "정확히 3줄, 각 줄 80자 이내" 같은 제약 명시)

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    다음 중 하나를 추가하세요. (1) "민원" 카테고리에도 별도 프롬프트로 요약 후 Slack 전달, (2) 첨부파일이 있으면 파일명도 요약에 포함, (3) 요약 결과를 Google Sheets에 누적 기록해 일일 통계 생성.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    같은 "분류 후 요약" 구조는 다른 입력에도 그대로 쓰입니다.

    - Google Form 민원 접수 → 카테고리 분류 → 담당자 통보
    - Slack 채널 메시지 → 중요/잡담 분류 → 중요만 주간 리포트

    현재 워크플로우를 복제(Duplicate)하고 입력 노드만 바꿔보세요.
