# Project 01. 보도자료 초안 작성 도우미

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 보도자료 초안 작성 도우미 |
| 소요 시간 | 1교시 (50분) |
| 선행 Lab | **Lab 02 · 프롬프트 템플릿 민원 답변**, **Lab 03 · 변수로 입력 폼 만들기** |
| 사용 도구 | Dify Cloud (Chatbot), ChatGPT |

📌 **진행 흐름**

`Phase 1 정의하기 (약 10분)` → `Phase 2 구현하기 (약 28분)` → `Phase 3 확장하기 (약 12분)`

> 이 프로젝트는 **Lab 02 · 프롬프트 템플릿 민원 답변**, **Lab 03 · 변수로 입력 폼 만들기**에서 익힌 시스템 프롬프트 설계와 변수 입력 폼 구성을 통합 적용합니다.

### 시나리오

> 당신은 지자체 홍보 담당 주무관입니다.
> 매주 부서별로 사업·행사 보도자료 요청이 들어오는데, 담당자마다 제목·리드·본문 구조가 제각각이고 육하원칙(누가/언제/어디서/무엇을/왜/어떻게)이 빠진 경우가 잦습니다. 당신은 "최소 정보만 넣으면 표준 구조의 초안을 만들어주는 사내용 도우미"를 Dify로 만들어 부서 공용 링크로 배포하려 합니다.

### 최종 산출물

- 작동하는 Dify Chatbot 앱 (공유 링크)
- 변수 5~6개가 입력 폼으로 연결된 보도자료 생성 챗봇
- 3개 이상의 샘플 입력으로 생성한 초안 예시

---

## Phase 1. 정의하기 (Define) — 약 10분

> 무엇을 해결할 것인지 정리하는 단계입니다. 바로 만들기 시작하지 말고, 먼저 문제를 이해하세요.
>
> Lab에서 시도한 프롬프트 템플릿과 변수 입력 폼을 바탕으로, 지금부터는 스스로 문제를 정의합니다.

!!! abstract "과제"
    보도자료에 반드시 들어가야 할 입력 변수와, 생성될 보도자료의 표준 구조·톤앤매너 원칙을 정리하세요.

**요구사항 정리**

- [ ] 입력 변수 5~6개가 정의되었다 (사업명 · 일시 · 장소 · 대상 · 주요내용 · 기대효과)
- [ ] 보도자료 표준 구조 4단(제목 / 리드 1문단 / 본문 2~3문단 / 맺음말)이 명시되었다
- [ ] 톤앤매너 3원칙(객관적 3인칭 · 간결체 · 구체 수치 우선)이 문장으로 정리되었다
- [ ] 금지 표현 목록(과장 수식어 · 주관적 표현)이 작성되었다

**활용 도구 & 데이터**

- Dify Chatbot 앱 (Cloud)
- ChatGPT: 표준 구조·톤 원칙을 정리할 때 보조
- 참고 자료: 소속 기관 최근 보도자료 2~3건 (구조 분석용)

??? tip "힌트 1: 어디서 시작할지 모르겠다면"
    "이 보도자료를 받는 기자가 처음 30초 안에 알고 싶은 정보는 무엇인가?"를 출발점으로 삼으세요. 대개 누가·무엇을·언제·어디서 네 가지가 리드 문단에 들어갑니다.

    최근 공개된 내 기관 보도자료 2건을 펼쳐 놓고, 제목·첫 문단·마지막 문단에서 공통적으로 등장하는 요소를 체크리스트로 만들어보세요.

??? tip "힌트 2: 요구사항을 정리하기 어렵다면"
    ChatGPT에게 이렇게 물어보세요:
    "나는 지자체 홍보 담당이고, 부서 직원들이 최소 정보만 넣으면 표준 보도자료 초안이 만들어지는 도우미를 만들려 합니다. 필수 입력 항목 5~6개와 피해야 할 표현 5가지를 추천해주세요."

    AI 제안을 그대로 쓰지 말고, 내 기관 보도자료 스타일에 맞게 추리고 재작성하세요.

## Phase 2. 구현하기 (Build) — 약 28분

> 정의한 내용을 바탕으로 직접 만들어보는 단계입니다. 처음부터 완벽할 필요는 없습니다. 작동하는 첫 번째 버전을 목표로 하세요.

!!! abstract "과제"
    Phase 1에서 정의한 입력 변수·구조·톤 원칙을 Dify Chatbot 앱에 구현하고, 샘플 입력으로 실제 보도자료 초안이 생성되는지 확인하세요.

**마일스톤**

☐ Dify에서 새 Chatbot 앱을 만들고 모델을 선택했다
☐ Phase 1의 5~6개 변수를 입력 폼(Variables)으로 등록했다
☐ 시스템 프롬프트에 표준 구조 + 톤 원칙 + 금지 표현을 모두 반영했다
☐ 3개 이상의 서로 다른 사업 시나리오 샘플로 초안을 생성해 구조 준수 여부를 확인했다
☐ 동료 1명에게 공유 링크를 보내 테스트 입력을 받고 피드백을 수집했다

**활용 도구 & 데이터**

- Dify Chatbot 앱: Variables, System Prompt, Preview 기능
- ChatGPT: 시스템 프롬프트 초안 다듬기, 톤 체크
- 샘플 사업 3건 (행사·공모·정책 각 1건 권장)

??? tip "힌트 1: 구현 방향이 잡히지 않는다면"
    앱을 한 번에 완성하려 하지 말고 단계적으로 만들어보세요:

    1. 변수 2개(사업명 · 주요내용)만 먼저 넣고 가장 짧은 프롬프트로 결과 확인
    2. 구조 지시문(제목 → 리드 → 본문 → 맺음말)만 추가해 결과 비교
    3. 톤 원칙 추가, 금지 표현 추가 순으로 한 줄씩 늘려가기

    각 단계마다 같은 입력을 넣고 결과가 어떻게 개선되는지 눈으로 확인하세요.

??? tip "힌트 2: 도구 활용이 막힌다면"
    Dify에서 변수는 `{{사업명}}` 형태로 시스템 프롬프트 안에 삽입합니다. Lab 03에서 만든 입력 폼 예시를 다시 열어 변수 선언 → 프롬프트 삽입 → Preview 흐름을 복습하세요.

    톤이 생각대로 안 나오면 ChatGPT에 "아래 프롬프트를 공공기관 보도자료 톤(3인칭, 간결체)에 맞게 다듬어줘" 하고 시스템 프롬프트 자체를 리팩터링 받을 수 있습니다.

??? tip "힌트 3: 구체적인 예시가 필요하다면"
    시스템 프롬프트 뼈대 예시:

    ```
    당신은 공공기관의 홍보 담당자입니다.
    아래 입력 변수를 바탕으로, 다음 구조의 보도자료 초안을 작성하세요.

    [구조]
    1. 제목 (1줄, 핵심 사실 + 수치)
    2. 리드 문단 (누가·무엇을·언제·어디서, 3문장 이내)
    3. 본문 2~3문단 (배경 → 주요 내용 → 기대효과)
    4. 맺음말 (담당자 인용 또는 향후 계획)

    [톤 원칙]
    - 객관적 3인칭, 간결체, 구체 수치 우선
    - 다음 표현은 사용하지 말 것: "획기적", "최고의", "매우 ~한"

    [입력]
    사업명: {{사업명}}
    일시: {{일시}}
    장소: {{장소}}
    대상: {{대상}}
    주요내용: {{주요내용}}
    기대효과: {{기대효과}}
    ```

    이 뼈대를 그대로 붙여넣고 Preview에서 실행해 본 뒤, 내 기관 스타일에 맞게 줄 단위로 수정하세요.

## Phase 3. 확장하기 (Extend) — 약 12분

> 만든 결과물을 돌아보고, 더 나아가는 단계입니다. 아래 두 가지 트랙 중 원하는 방향을 선택하세요.

### 트랙 A: 개선 루프

Phase 2의 결과물을 스스로 평가하고, 부족한 부분을 개선합니다. 개선 후 다시 Phase 2로 돌아가 반복할 수 있습니다.

!!! abstract "과제"
    서로 다른 3개 샘플로 초안을 생성해 읽어본 뒤, 부자연스러운 문장 또는 누락된 정보 3곳을 찾아 시스템 프롬프트를 수정하세요.

??? tip "힌트: 무엇을 개선할지 모르겠다면"
    다음 관점으로 점검하세요:

    - "리드 문단에서 누가·무엇을·언제·어디서가 모두 확인되는가?"
    - "금지 표현이 여전히 등장하는가?"
    - "기대효과가 구체적 수치·대상으로 서술되는가, 아니면 추상적 표현인가?"

    발견한 문제를 **프롬프트의 어느 줄을 어떻게 바꿔야 해결되는지** 한 줄 코멘트로 기록한 뒤 수정하세요.

### 트랙 B: 기능 확장

만든 결과물을 다른 상황이나 시나리오에 추가로 적용해봅니다.

!!! abstract "과제"
    작동하는 챗봇에 다음 중 하나를 추가해보세요: (1) "회의록 → 보도자료" 입력 경로(회의록 붙여넣기 시 자동 변수 추출), (2) 영문 보도자료 병기, (3) "짧은 SNS 카드뉴스용 문구" 출력 옵션.

??? tip "힌트: 어떤 상황에 확장할지 모르겠다면"
    독자가 바뀌면 톤·길이·채널이 모두 바뀝니다. 예시:

    - 국내 기자 + 공식 배포 → 지금의 보도자료 형식
    - SNS 이용자 + 카드뉴스 → 150자 이내, 이모지 활용 가능
    - 해외 기관 + 영문 → 간결한 5W1H + 인용문

    가장 쉬운 확장(예: 영문 병기 옵션)부터 시도하세요. 시스템 프롬프트에 조건 분기 한 줄만 추가해도 구현됩니다.
